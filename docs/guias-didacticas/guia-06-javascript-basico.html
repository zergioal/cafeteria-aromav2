Guía 06 – JavaScript Básico: Programando el Cerebro de Aroma

Módulo 3: JavaScript | Del HTML estático a la web interactiva

Nivel: Principiante Tiempo estimado: 6 horas Prerequisito: Guías 01-05

Objetivos de aprendizaje

  • Comprender qué es JavaScript y para qué sirve en un proyecto web real.
  • Declarar y usar variables con let y const.
  • Conocer los tipos de datos fundamentales: string, number, boolean, array y object.
  • Aplicar operadores aritméticos, de comparación y lógicos.
  • Trabajar con strings, arrays y objetos usando sus métodos más importantes.
  • Escribir condicionales, bucles y funciones.
  • Entender cómo se usa JavaScript en el proyecto Cafetería Aroma.

1. ¿Qué es JavaScript?

Analogía de la cafetería: Si el HTML es el esqueleto (la estructura del local: mesas, mostrador, paredes) y el CSS es la decoración y el estilo (colores, iluminación, uniformes), entonces JavaScript son los músculos y el cerebro. JavaScript hace que los empleados se muevan, que la caja registradora calcule el total, que el menú digital responda cuando tocas un plato. Sin JavaScript, la web es un cartel estático; con JavaScript, es una aplicación interactiva.

¿Qué puede hacer JavaScript en Aroma?

  • Mostrar u ocultar elementos: el panel de admin solo aparece si el usuario es administrador.
  • Hacer cálculos: calcular el total del carrito sumando precios y cantidades.
  • Guardar datos temporalmente: mantener el carrito en localStorage aunque se recargue la página.
  • Comunicarse con el servidor: pedir la lista de productos al backend cuando se abre el menú.
  • Responder a acciones del usuario: mostrar una notificación al agregar un producto, cambiar de pestaña al hacer clic en la navbar.
  • Validar formularios: verificar que el email tiene el formato correcto antes de enviarlo.

¿Dónde se ejecuta?

JavaScript puede correr en dos lugares distintos:

  • En el navegador del usuario (client-side): es lo que estudiamos en esta guía. El código lo descarga y ejecuta el propio navegador (Chrome, Firefox, Edge).
  • En el servidor (server-side): usando Node.js, JavaScript también puede correr en el servidor para procesar peticiones, acceder a bases de datos, etc. El backend de Aroma usa Node.js + Express.
Buena práctica: En Aroma cargamos los archivos JavaScript al final del <body> (o con el atributo defer) para que el HTML se muestre rápido y JS se cargue después sin bloquear la página.

2. Variables – Las cajas de almacenamiento

Analogía: Una variable es como una caja con una etiqueta. La etiqueta es el nombre, y adentro puedes guardar cualquier cosa: un número, un texto, una lista de productos. Cuando necesitas lo que guardaste, buscas la caja por su nombre.

En JavaScript moderno (ES6+) hay tres formas de declarar variables. En Aroma usamos principalmente const y let:

// Tres formas de declarar variables
var nombreAntiguo = "Juan";        // Antigua, evitar
let nombreModificable = "María";   // Puede cambiar
const nombreFijo = "Carlos";       // No puede cambiar (constante)

// En Aroma usamos principalmente const y let
const API_URL = "https://mi-backend.com"; // Nunca cambia → const
let carrito = [];                          // Puede cambiar → let
Regla de oro: Usa siempre const por defecto. Solo cambia a let si sabes que el valor va a cambiar. Nunca uses var en código nuevo.

Reglas para nombrar variables

  • Usa camelCase: nombreProducto, totalCarrito, estaActivo.
  • No pueden iniciar con un número: 1producto es inválido; producto1 sí.
  • No uses palabras reservadas de JavaScript: let, const, function, return, etc.
  • Sé descriptivo: p es malo; precio es bueno; precioConDescuento es excelente.
  • Las constantes globales de configuración se escriben en MAYÚSCULAS con guiones bajos: API_URL, MAX_ITEMS.

3. Tipos de Datos

JavaScript es un lenguaje de tipado dinámico: una misma variable puede guardar distintos tipos de datos. Es importante conocerlos todos para no cometer errores al operar con ellos.

// String (texto)
const nombre = "Café Latte";

// Number (número)

// Boolean (verdadero/falso)

const estaEnCarrito = false;

// Array (lista)
const categorias = ["Cafés", "Jugos", "Pasteles", "Sándwiches"];

// Object (objeto con propiedades)

4. Operadores

// Aritméticos
const suma = 10 + 5;          // 15
const resta = 10 - 3;         // 7
const multiplicacion = 4 * 5; // 20
const division = 15 / 3;      // 5
const modulo = 17 % 5;        // 2 (residuo de la división)
const potencia = 2 ** 8;      // 256

// Asignación compuesta
let total = 0;
total += 18.50;  // total = total + 18.50  → 18.50
total -= 5;      // total = total - 5      → 13.50
total *= 2;      // total = total * 2      → 27

// Comparación
5 === 5      // true  (igual valor Y tipo - usar SIEMPRE este)
5 == "5"     // true  (igual valor, ignora tipo - EVITAR)
5 !== 3      // true  (diferente valor o tipo)
10 > 5       // true  (mayor que)
10 >= 10     // true  (mayor o igual)
3 < 8        // true  (menor que)

// Lógicos
true && true   // true  (AND - ambos deben ser true)
true || false  // true  (OR - al menos uno true)
!true          // false (NOT - invierte el valor)

// En Aroma: ejemplos reales
const puedeOrdenar = usuarioLogueado && carritoNoVacio;
const mostrarLogin = !usuarioLogueado;
Usa siempre === en lugar de ==: el operador == hace conversiones de tipo automáticas que pueden generar bugs difíciles de encontrar. Por ejemplo, 0 == false es true, lo cual casi nunca es lo que quieres. Con === (igualdad estricta) no hay sorpresas.

5. Strings – Trabajando con Texto

Los strings son el tipo de dato que más vas a usar para construir la interfaz de Aroma: nombres de productos, descripciones, mensajes al usuario, HTML dinámico.

const nombre = "Café Latte";

// Propiedades y métodos de strings
nombre.length             // 10 (cantidad de caracteres)
nombre.toUpperCase()      // "CAFÉ LATTE"
nombre.toLowerCase()      // "café latte"
nombre.includes("Latte")  // true
nombre.startsWith("Café") // true
nombre.trim()             // Elimina espacios al inicio y al fin
nombre.replace("Latte", "Americano") // "Café Americano"

// Template literals (muy usados en Aroma)
const precio = 18.50;
const html = `
    <div class="producto">
        <h3>${nombre}</h3>
        <span>Bs. ${precio.toFixed(2)}</span>
    </div>
`;

// toFixed(2) redondea a 2 decimales → "18.50"
console.log(precio.toFixed(2));  // "18.50"
console.log((18.5).toFixed(2));  // "18.50"
console.log((18).toFixed(2));    // "18.00"

// Concatenación (alternativa a template literals)
const mensaje = "El producto " + nombre + " cuesta Bs. " + precio;
Prefiere los template literals: el backtick (`) permite incrustar variables directamente con ${variable} y escribir strings en múltiples líneas. Es mucho más legible que concatenar con +.

6. Arrays – Listas de Datos

Los arrays son fundamentales en Aroma: la lista de productos, el carrito, las categorías, los pedidos... todo se maneja con arrays.

// Crear arrays
const frutas = ["manzana", "banana", "naranja"];
const numeros = [1, 2, 3, 4, 5];
const mixto = ["texto", 42, true, null]; // Puede mezclar tipos

// Acceder por índice (empieza en 0, no en 1)
frutas[0]     // "manzana"
frutas[1]     // "banana"
frutas[2]     // "naranja"
frutas.length // 3

// push - agregar al final
const carrito = [];
carrito.push({ nombre: "Café Latte", precio: 18 });
carrito.push({ nombre: "Jugo de Naranja", precio: 12 });
// carrito ahora tiene 2 elementos

// pop - quitar el último elemento y devolverlo
const ultimo = carrito.pop();

// find - encontrar el PRIMER elemento que cumple la condición
const producto = carrito.find(item => item.nombre === "Café Latte");

// filter - crear nuevo array con los que cumplen la condición
const cafes = productos.filter(p => p.categoria === "Cafés");
const activos = productos.filter(p => p.activo === true);

// map - crear nuevo array transformando cada elemento
const nombres = productos.map(p => p.nombre);
const htmlItems = productos.map(p => `<li>${p.nombre}: Bs.${p.precio}</li>`);

// forEach - recorrer sin devolver nada nuevo
productos.forEach(producto => {
    console.log(producto.nombre);
});

// reduce - acumular un valor (ej: suma total)
const total = carrito.reduce((acum, item) => acum + item.precio, 0);
// Empieza con acum = 0, suma cada item.precio

// includes - verificar si un valor está en el array
const tieneLeche = ingredientes.includes("leche");

// splice - eliminar o insertar por índice
carrito.splice(1, 1); // Eliminar 1 elemento en la posición 1

// findIndex - encontrar el índice de un elemento
const indice = carrito.findIndex(item => item.id === "abc-123");
Analogía del menú de pizarrón: Un array es como la pizarra de productos disponibles en la cafetería. Cada posición (índice) tiene un producto. Puedes agregar al final (push), quitar el último (pop), buscar uno específico (find), o crear una nueva pizarra solo con los cafés (filter).
Métodos que NO modifican el array original: filter, map, find, findIndex, reduce devuelven un nuevo valor sin tocar el original. En cambio, push, pop y splice SÍ modifican el array original.

7. Objetos – Agrupando Propiedades Relacionadas

Un objeto agrupa datos relacionados bajo un mismo nombre. En Aroma, cada producto, cada pedido y cada usuario es un objeto.

// Crear objeto
const producto = {
    id: "abc-123",
    nombre: "Café Latte",
    precio: 18.50,
    categoria: "Cafés",
    activo: true
};

// Acceder a propiedades
producto.nombre      // "Café Latte" (notación punto - más común)
producto["precio"]   // 18.50 (notación corchete - útil con variables)

// Modificar propiedades
producto.precio = 20;
producto.activo = false;

// Agregar nueva propiedad
producto.imagen = "cafe-latte.jpg";

// Eliminar propiedad
delete producto.imagen;

// Verificar si existe una propiedad
"nombre" in producto              // true
producto.hasOwnProperty("activo") // true

// Destructuring (muy usado en Aroma - extrae propiedades en variables)
const { nombre, precio, categoria } = producto;
// Es exactamente igual a escribir:
// const nombre = producto.nombre;
// const precio = producto.precio;
// const categoria = producto.categoria;

// Spread operator - copiar y modificar
const productoActualizado = { ...producto, precio: 22 };
// Copia TODAS las propiedades de producto, pero precio vale 22

// Object.keys(), values(), entries()
Object.keys(producto)    // ["id", "nombre", "precio", "categoria", "activo"]
Object.values(producto)  // ["abc-123", "Café Latte", 18.50, "Cafés", true]
Object.entries(producto) // [["id","abc-123"], ["nombre","Café Latte"], ...]
Ojo con la notación corchete: es útil cuando el nombre de la propiedad está en una variable. Por ejemplo: const clave = "precio"; producto[clave]. Esto devuelve 18.50. Con notación punto no se puede hacer esto.

8. Condicionales – Tomando Decisiones

// if / else if / else
const rol = "cashier";

if (rol === "admin") {
    console.log("Puede hacer todo");
} else if (rol === "cashier") {
    console.log("Puede gestionar pedidos");
} else {
    console.log("Solo puede ver el menú");
}

// Operador ternario (if/else en una línea)
const mensaje = usuarioLogueado ? "Bienvenido" : "Por favor inicia sesión";
const clase = estaActivo ? "btn-activo" : "btn-inactivo";

// Nullish coalescing (??) - usar valor por defecto si es null o undefined
const nombre = usuario?.nombre ?? "Invitado";
// Si usuario.nombre es null o undefined → usa "Invitado"
// Si tiene un valor real (incluso 0 o false) → lo usa

// Optional chaining (?.) - acceder sin error si algo es null
const rolUsuario = usuario?.perfil?.rol;
// Si usuario es null/undefined → devuelve undefined (no lanza error)
// Si usuario existe pero perfil no → devuelve undefined (no lanza error)

// switch - para múltiples casos del mismo valor
switch (metodoPago) {
    case "qr":
        mostrarQR();
        break;
    case "efectivo":
        mostrarPagoEfectivo();
        break;
    default:
        mostrarError("Método no válido");
}
Cuándo usar el ternario: úsalo solo cuando la condición es simple y el resultado es corto. Si el bloque if tiene más de 2-3 líneas, escribe el if/else completo para que sea más legible.

9. Bucles (Loops) – Repitiendo Acciones

Los bucles permiten ejecutar código varias veces sin repetirlo manualmente. Son esenciales para recorrer listas de productos, items del carrito, etc.

// for clásico - cuando sabes cuántas veces repetir
for (let i = 0; i < 5; i++) {
    console.log(`Iteración ${i}`); // 0, 1, 2, 3, 4
}

// for...of - recorrer arrays (el más usado en Aroma)
const productos = ["Café", "Jugo", "Pastel"];
for (const producto of productos) {
    console.log(producto);
    // Imprime: Café, luego Jugo, luego Pastel
}

// for...in - recorrer las propiedades (claves) de un objeto
const config = { url: "...", key: "...", timeout: 5000 };
for (const clave in config) {
    console.log(`${clave}: ${config[clave]}`);
    // url: ...
    // key: ...
    // timeout: 5000
}

// while - cuando no sabes cuántas veces repetir
let intentos = 0;
while (intentos < 3) {
    // intentar conexión con el servidor
    intentos++;
}
En Aroma usamos casi siempre los métodos de array (forEach, map, filter) en lugar de bucles for clásicos. Son más expresivos y fáciles de leer. Pero el for...of sigue siendo muy útil cuando necesitas usar await dentro del bucle.

10. Funciones – Bloques de Código Reutilizables

Las funciones son la unidad fundamental de organización del código. En lugar de repetir la misma lógica en varios lugares, la encapsulas en una función con un nombre descriptivo.

// Declaración de función (hoisting - se puede llamar antes de definirla)
function calcularTotal(items) {
    let total = 0;
    for (const item of items) {
        total += item.precio * item.cantidad;
    }
    return total;
}

// Llamar a la función
const miTotal = calcularTotal(carrito); // Devuelve un número

// Expresión de función (NO tiene hoisting)
const calcularDescuento = function(total, porcentaje) {
    return total * (porcentaje / 100);
};

// Arrow function (ES6 - la más usada en Aroma)
const formatearPrecio = (precio) => `Bs. ${precio.toFixed(2)}`;
const sumar = (a, b) => a + b;
const saludar = nombre => `Hola, ${nombre}`; // Un parámetro: sin paréntesis

// Función con parámetros por defecto
function crearPedido(cliente, metodo = "efectivo", notas = "") {
    return { cliente, metodo, notas };
    // shorthand: { cliente: cliente } → { cliente }
}
crearPedido("Ana");                   // metodo="efectivo", notas=""
crearPedido("Ana", "qr", "Sin azúcar"); // todos los parámetros

// Función que devuelve un objeto
function crearItemCarrito(producto, cantidad) {
    return {
        id: producto.id,
        nombre: producto.nombre,
        precio: producto.precio,
        cantidad: cantidad,
        subtotal: producto.precio * cantidad
    };
}
Analogía: Una función es como una receta de cocina. Le das los ingredientes (parámetros), sigue los pasos, y te devuelve el resultado (return). Puedes usar la misma receta para hacer café para 1 persona o para 10, solo cambian los ingredientes.

Aplicando en Aroma: El Módulo config.js

Archivo real del proyecto: frontend/js/config.js

Este es el primer archivo JavaScript que se carga en Aroma. Define las constantes de configuración que todos los demás módulos necesitan.

// frontend/js/config.js
const CONFIG = {
    SUPABASE_URL: 'https://xxxxx.supabase.co',
    SUPABASE_ANON_KEY: 'eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...',
    API_BASE_URL: 'http://localhost:3000'
};

Analicemos cada decisión de este pequeño pero importante archivo:

  • ¿Por qué const CONFIG y no let? Porque el objeto de configuración nunca se reasigna a un valor diferente. const previene que accidentalmente escribas CONFIG = otraCosa. Nota: las propiedades internas sí podrían cambiar, pero en este caso no queremos eso.
  • ¿Por qué CONFIG en mayúsculas? Es una convención universal: las variables que son constantes de configuración global se escriben en SNAKE_CASE_MAYUSCULAS. Así, cualquier desarrollador que lea el código sabe inmediatamente que ese valor no debería cambiar durante la ejecución.
  • SUPABASE_URL y SUPABASE_ANON_KEY: son las credenciales para conectarse a la base de datos Supabase (el servicio de backend de Aroma). La anon_key es pública (está diseñada para usarse en el cliente), por eso no es un secreto ponerla en el frontend.
  • API_BASE_URL: la dirección del servidor Express de Aroma. En desarrollo es localhost:3000; en producción sería la URL real del servidor.
¿Por qué un objeto y no variables separadas? Agrupar la configuración en un objeto CONFIG tiene varias ventajas: evita contaminar el espacio global con muchas variables sueltas, deja claro que esos valores van juntos, y se puede pasar el objeto completo a funciones que lo necesiten.

Resumen de la Guía 06

  • JavaScript es el lenguaje que da comportamiento a la web. Se ejecuta en el navegador (y también en el servidor con Node.js).
  • Usa const por defecto; solo usa let cuando el valor necesita cambiar. Nunca uses var.
  • Los 7 tipos de datos fundamentales: string, number, boolean, null, undefined, array y object.
  • Usa siempre === (igualdad estricta) en lugar de == para comparar.
  • Los template literals con backticks (`) son la forma moderna y legible de construir strings con variables.
  • Los métodos de array más importantes en Aroma: push, filter, map, find, findIndex, reduce, forEach.
  • El destructuring y el spread operator (...) simplifican el trabajo con objetos y arrays.
  • Las arrow functions son la forma más concisa y moderna de escribir funciones.
  • JSON.stringify() convierte objetos a texto; JSON.parse() hace lo inverso.
  • El archivo config.js de Aroma centraliza las constantes de configuración usando un objeto const CONFIG = {...}.
