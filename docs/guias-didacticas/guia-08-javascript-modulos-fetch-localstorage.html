1. localStorage – Guardar Datos en el Navegador

Analogía: Imagina que cada sitio web tiene una pequeña libreta personal que el navegador guarda en la computadora del usuario. Cuando el usuario cierra la ventana y vuelve después, el sitio puede leer esa libreta y recordar lo que guardó: qué productos hay en el carrito, si el usuario está logueado, qué idioma prefiere. Solo ese sitio web puede leer su propia libreta; otros sitios no tienen acceso.

localStorage es un espacio de almacenamiento en el navegador que persiste incluso después de cerrar y reabrir el navegador (a diferencia de sessionStorage, que se borra al cerrar la pestaña).

// Guardar un valor simple
localStorage.setItem('usuario', 'Ana');

// Leer ese valor después (incluso en otra sesión)
const nombre = localStorage.getItem('usuario'); // "Ana"

// IMPORTANTE: localStorage solo guarda strings
// Para objetos y arrays, debes convertirlos con JSON

// Guardar un array de objetos
const carrito = [
    { id: 1, nombre: "Café Latte", precio: 18, cantidad: 2 },
    { id: 2, nombre: "Jugo de Naranja", precio: 12, cantidad: 1 }
];
localStorage.setItem('carrito', JSON.stringify(carrito));

// Leer y convertir de vuelta a objeto JavaScript
const carritoGuardado = localStorage.getItem('carrito'); // String JSON
const carritoParsed = JSON.parse(carritoGuardado);       // Array de objetos

// Eliminar un item específico
localStorage.removeItem('usuario');

// Limpiar TODO el localStorage (¡cuidado!)
localStorage.clear();

// Patrón seguro para leer (protege contra JSON malformado o clave inexistente)
function leerDelStorage(clave, valorPorDefecto = null) {
    try {
        const guardado = localStorage.getItem(clave);
        return guardado ? JSON.parse(guardado) : valorPorDefecto;
    } catch (error) {
        console.error('Error leyendo localStorage:', error);
        return valorPorDefecto;
    }
}

2. El Módulo Cart de Aroma (cart.js)

Archivo real del proyecto: frontend/js/cart.js

Este módulo implementa toda la lógica del carrito de compras. Es un excelente ejemplo del Module Pattern combinado con localStorage. Lee cada método con atención.

// frontend/js/cart.js
const Cart = {
    // Clave única para identificar los datos en localStorage
    STORAGE_KEY: 'aroma_cart',

    // Obtener el carrito actual desde localStorage
    getItems() {
        try {
            const guardado = localStorage.getItem(this.STORAGE_KEY);
            return guardado ? JSON.parse(guardado) : [];
        } catch {
            return []; // Si el JSON está corrupto, devolvemos array vacío
        }
    },

    // Guardar el carrito en localStorage y actualizar el badge
    saveItems(items) {
        localStorage.setItem(this.STORAGE_KEY, JSON.stringify(items));
        this.updateBadge(); // Siempre actualizar el contador visual
    },

    // Agregar un producto (o incrementar cantidad si ya existe)
    addItem(producto, cantidad = 1) {
        const items = this.getItems();
        const indice = items.findIndex(item => item.id === producto.id);

        if (indice >= 0) {
            // El producto ya está en el carrito: solo sumamos cantidad
            items[indice].cantidad += cantidad;
        } else {
            // Producto nuevo: crear el objeto y agregarlo
            items.push({
                id: producto.id,
                nombre: producto.nombre,
                precio: producto.precio,
                imagen: producto.imagen_url,
                cantidad: cantidad
            });
        }

        this.saveItems(items);
        mostrarToast(`${producto.nombre} agregado al carrito`, 'success');
    },

    // Eliminar un producto completo del carrito
    removeItem(productoId) {
        // filter devuelve todos los items EXCEPTO el que tiene ese id
        const items = this.getItems().filter(item => item.id !== productoId);
        this.saveItems(items);
    },

    // Cambiar la cantidad de un producto específico
    updateQuantity(productoId, nuevaCantidad) {
        if (nuevaCantidad <= 0) {
            // Si la cantidad llega a 0 o menos, eliminar el producto
            this.removeItem(productoId);
            return;
        }

        // map: recorrer todos los items y modificar solo el que corresponde
        const items = this.getItems().map(item =>
            item.id === productoId
                ? { ...item, cantidad: nuevaCantidad } // Spread + override
                : item                                  // Los demás sin cambio
        );
        this.saveItems(items);
    },

    // Calcular el precio total del carrito
    getTotal() {
        return this.getItems().reduce(
            (total, item) => total + (item.precio * item.cantidad),
            0 // Valor inicial del acumulador
        );
    },

    // Contar la cantidad total de unidades en el carrito
    getCount() {
        return this.getItems().reduce(
            (total, item) => total + item.cantidad,
            0
        );
    },

    // Vaciar el carrito completamente
    clear() {
        localStorage.removeItem(this.STORAGE_KEY);
        this.updateBadge();
    },

    // Actualizar el número rojo (badge) sobre el ícono del carrito en la navbar
    updateBadge() {
        const badge = document.getElementById('cartBadge');
        if (badge) {
            const count = this.getCount();
            badge.textContent = count;
            badge.style.display = count > 0 ? 'flex' : 'none';
        }
    }
};

Analizando las decisiones de diseño

El patrón "obtener → modificar → guardar"

Todos los métodos que modifican el carrito siguen la misma estructura: (1) leer el estado actual con getItems(), (2) crear la nueva versión del array, (3) guardar con saveItems(). Esto garantiza que el localStorage siempre tenga el estado más reciente.

Inmutabilidad con map y spread operator

En updateQuantity, en lugar de modificar el objeto directamente (item.cantidad = nuevaCantidad), creamos un nuevo objeto con el spread: { ...item, cantidad: nuevaCantidad }. Esto es la programación inmutable: nunca mutamos los datos originales, creamos versiones nuevas. Es más predecible y fácil de depurar.

¿Por qué STORAGE_KEY como constante del objeto?

Si mañana necesitas cambiar la clave de localStorage (por ejemplo, para evitar conflictos con otra aplicación), solo cambias un lugar: la definición de STORAGE_KEY. Si hubieras escrito 'aroma_cart' repetido en 5 métodos, tendrías que cambiar 5 lugares y es fácil olvidar uno.

3. Promises – La Programación Asíncrona

Analogía de la pizzería: Cuando llamas a una pizzería y pides una pizza, no te quedas paralizado en la silla esperando a que llegue. Te dan un número de pedido (una promesa) y sigues con tu vida. Cuando llegue la pizza (la promesa se resolvió), el repartidor toca el timbre y recibes el resultado. Si hubo un problema (se quemó la pizza → la promesa fue rechazada), te llaman para decirte que no puede entregarse.

En JavaScript, muchas operaciones son asíncronas: tardan tiempo y no deben bloquear todo lo demás mientras esperan. Pedir datos a un servidor puede tardar 200ms, 500ms o más. Una Promise representa un valor que todavía no está disponible pero estará en algún momento (o fallará).

// Una Promise puede estar en 3 estados:
// - pending (esperando): todavía no se resolvió ni rechazó
// - fulfilled (resuelta): la operación terminó con éxito
// - rejected (rechazada): la operación falló

// Crear una Promise manualmente (para entender cómo funciona)
const miPromesa = new Promise((resolve, reject) => {
    // Simulando una operación que tarda 1 segundo
    setTimeout(() => {
        const exito = true;
        if (exito) {
            resolve("¡Operación exitosa!");  // Resolver con un valor
        } else {
            reject("Hubo un error");         // Rechazar con un mensaje de error
        }
    }, 1000);
});

// Consumir la Promise con .then() y .catch()
miPromesa
    .then(resultado => {
        console.log(resultado); // "¡Operación exitosa!"
    })
    .catch(error => {
        console.error(error);   // Solo se ejecuta si fue rechazada
    })
    .finally(() => {
        console.log("Esto se ejecuta siempre, haya éxito o error");
    });

// Fetch API - la forma nativa de hacer peticiones HTTP
// fetch() devuelve una Promise que se resuelve con un objeto Response
fetch('https://mi-api.com/productos')
    .then(response => {
        // Primero verificar si la respuesta fue exitosa (código 200-299)
        if (!response.ok) {
            throw new Error(`Error HTTP: ${response.status}`);
        }
        return response.json(); // .json() también devuelve una Promise
    })
    .then(datos => {
        console.log(datos);          // El array de productos
        renderizarProductos(datos);
    })
    .catch(error => {
        console.error('Error al cargar:', error);
    });
¿Por qué dos .then()? fetch() resuelve cuando recibe los headers de la respuesta, no cuando tiene el cuerpo completo. response.json() lee el cuerpo y devuelve otra Promise. Por eso necesitas un segundo .then() para procesar los datos reales.

4. async/await – Promises que Parecen Código Síncrono

async/await es "azúcar sintáctica" sobre las Promises. Hace exactamente lo mismo que .then()/.catch(), pero el código se lee de arriba hacia abajo, como si fuera síncrono. Esto lo hace mucho más fácil de entender y de depurar.

// El mismo código: versión con .then() y versión con async/await

// VERSIÓN 1: con .then() y .catch()
function cargarProductos() {
    fetch('/api/products')
        .then(r => r.json())
        .then(datos => renderizarProductos(datos.products))
        .catch(err => mostrarError(err));
}

// VERSIÓN 2: con async/await (más legible)
async function cargarProductos() {
    try {
        const respuesta = await fetch('/api/products');
        const datos = await respuesta.json();
        renderizarProductos(datos.products);
    } catch (error) {
        mostrarError(error);
    }
}

// async: indica que la función va a contener operaciones asíncronas
// await: "espera aquí hasta que esta Promise se resuelva, luego continúa"

// Ejemplo más complejo con múltiples awaits
async function iniciarPaginaMenu() {
    try {
        // Esperar que carguen los productos
        const resProductos = await fetch('/api/products');
        const { products } = await resProductos.json();

        // Esperar que carguen las categorías
        const resCategorias = await fetch('/api/categories');
        const { categories } = await resCategorias.json();

        // Ahora tenemos ambos datos disponibles
        renderizarFiltros(categories);
        renderizarProductos(products);

    } catch (error) {
        console.error('Error al iniciar el menú:', error);
        mostrarToast('Error al cargar el menú', 'error');
    }
}

try/catch para manejo de errores

Con async/await, usamos try/catch en lugar de .catch(). El bloque try contiene el código que puede fallar; si falla, el error "cae" al bloque catch para manejarlo sin que la aplicación explote.

async function miOperacion() {
    try {
        // Código que puede lanzar un error
        const datos = await fetch('/api/algo');
        const json = await datos.json();
        // Si llega aquí, todo salió bien
        return json;
    } catch (error) {
        // Se ejecuta si CUALQUIER línea del try lanza un error
        console.error('Error:', error.message);
        // Puedes devolver un valor por defecto, mostrar un mensaje, etc.
        return null;
    } finally {
        // Se ejecuta SIEMPRE: tanto en éxito como en error
        // Útil para ocultar spinners de carga
        ocultarSpinner();
    }
}

5. El Módulo API de Aroma (api.js)

Archivo real del proyecto: frontend/js/api.js

Este módulo centraliza todas las peticiones HTTP al backend. En lugar de escribir fetch(...) con todos sus headers y opciones en cada lugar, tenemos un módulo que lo hace por nosotros de forma consistente.

// frontend/js/api.js
const Api = {
    // URL base del servidor (definida en config.js)
    BASE_URL: CONFIG.API_BASE_URL,

    // Obtener el token de autenticación desde localStorage
    getToken() {
        const session = localStorage.getItem('aroma_session');
        if (!session) return null;
        const { access_token } = JSON.parse(session);
        return access_token;
    },

    // Función base que hace la petición HTTP con todos los headers correctos
    async request(endpoint, opciones = {}) {
        const url = `${this.BASE_URL}${endpoint}`;
        const token = this.getToken();

        // Construir los headers, empezando con los por defecto
        const headers = {
            'Content-Type': 'application/json', // Indicamos que enviamos JSON
            ...opciones.headers                  // Spread: agrega headers extra si los hay
        };

        // Si el usuario está logueado, agregar el token de autenticación
        if (token) {
            headers['Authorization'] = `Bearer ${token}`;
        }

        // Hacer la petición
        const respuesta = await fetch(url, {
            ...opciones,  // Spread: copia method, body, etc. de opciones
            headers       // Pero siempre usa nuestros headers (sobreescribe los de opciones)
        });

        // Si el código HTTP no es 2xx, lanzar un error con el mensaje del servidor
        if (!respuesta.ok) {
            const error = await respuesta.json().catch(() => ({}));
            throw new Error(error.message || `Error ${respuesta.status}`);
        }

        return respuesta.json(); // Devolver el JSON de la respuesta
    },

    // Métodos de conveniencia que usan request() internamente

    get(endpoint) {
        return this.request(endpoint, { method: 'GET' });
    },

    post(endpoint, datos) {
        return this.request(endpoint, {
            method: 'POST',
            body: JSON.stringify(datos) // Convertir objeto a string JSON
        });
    },

    put(endpoint, datos) {
        return this.request(endpoint, {
            method: 'PUT',
            body: JSON.stringify(datos)
        });
    },

    patch(endpoint, datos) {
        return this.request(endpoint, {
            method: 'PATCH',
            body: JSON.stringify(datos)
        });
    }
};

Explicación de las decisiones clave

¿Qué es Authorization: Bearer token?

Cuando el servidor necesita saber quién está haciendo la petición (para verificar permisos), el cliente incluye un token en el header de autorización. El formato estándar es:

Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...

"Bearer" significa "portador": quien tenga este token, tiene los permisos asociados a él. El servidor lo verifica y sabe que la petición viene de un usuario autenticado con cierto rol (admin, cashier, customer).

¿Qué hace el spread operator ...opciones?

Permite pasar opciones adicionales a fetch() de forma flexible. Si llamas Api.post('/api/orders', datos), las opciones son { method: 'POST', body: ... }. El spread las copia al objeto de configuración de fetch. Es como mezclar dos objetos donde las propiedades de la derecha sobreescriben las de la izquierda si hay conflicto.

¿Por qué verificamos respuesta.ok?

La Promise de fetch() solo se rechaza si hay un error de red (sin conexión, DNS falla, etc.). Si el servidor responde con códigos de error como 404 Not Found o 500 Internal Server Error, la Promise igualmente se resuelve. Debemos verificar respuesta.ok (que es true solo para códigos 200-299) y lanzar manualmente el error si es necesario.

6. Arquitectura de Módulos del Proyecto

Aroma no usa el sistema de módulos ES6 (import/export) porque es JavaScript "vanilla" cargado directamente con etiquetas <script>. En cambio, cada módulo es un objeto global (const Cart = {...}, const Api = {...}) que está disponible en todos los archivos cargados después de él.

Orden de carga en el HTML (de arriba hacia abajo)

config.js → Define CONFIG (constantes globales)
auth.js → Define Auth (depende de CONFIG)
api.js → Define Api (depende de CONFIG)
cart.js → Define Cart (independiente)
pages/home.js → Define HomePage (usa Api)
pages/menu.js → Define MenuPage (usa Api, Cart)
pages/login.js → Define LoginPage (usa Auth, Api)
pages/cart-page.js → Define CartPage (usa Cart, Api)
pages/cashier.js → Define CashierPage (usa Api)
pages/admin.js → Define AdminPage (usa Api)
app.js → Define App (usa todo lo anterior)
El orden de carga importa: si api.js se cargara antes que config.js, la línea BASE_URL: CONFIG.API_BASE_URL fallaría porque CONFIG no existiría todavía. Siempre carga las dependencias antes que quienes las usan.

7. Cargando Productos desde la API (menu.js)

Archivo real del proyecto: frontend/js/pages/menu.js

Este es el módulo más completo de las páginas. Combina peticiones a la API, manejo de estado local, filtros, búsqueda y renderizado dinámico. Analiza el código línea por línea.

// frontend/js/pages/menu.js
const MenuPage = {
    // Estado local del módulo (datos que viven en memoria mientras la app está abierta)
    productos: [],      // Todos los productos cargados de la API
    categorias: [],     // Todas las categorías
    filtroActual: 'todos', // Categoría seleccionada actualmente
    busqueda: '',       // Texto de búsqueda actual

    // Inicializar la página del menú
    async init() {
        // Promise.all ejecuta ambas peticiones EN PARALELO (más rápido)
        // En vez de esperar que carguen categorías y LUEGO productos (secuencial),
        // los pedimos al mismo tiempo y esperamos a que AMBOS terminen
        await Promise.all([
            this.cargarCategorias(),
            this.cargarProductos()
        ]);

        // Solo cuando ambos están listos, configuramos la interactividad
        this.setupFiltros();
        this.setupBusqueda();
    },

    // Cargar la lista de productos desde el backend
    async cargarProductos() {
        try {
            // Mostrar spinner de carga mientras esperamos
            document.getElementById('productGrid').innerHTML =
                '<div class="loading">Cargando productos...</div>';

            // Petición al backend usando nuestro módulo Api
            const { products } = await Api.get('/api/products');

            // Guardar en el estado local del módulo
            this.productos = products;

            // Renderizar con los filtros aplicados
            this.renderizarProductos(this.productosFiltrados());

        } catch (error) {
            console.error('Error cargando productos:', error);
            document.getElementById('productGrid').innerHTML =
                '<div class="error">Error al cargar productos. Intenta de nuevo.</div>';
        }
    },

    // Cargar las categorías para los botones de filtro
    async cargarCategorias() {
        try {
            const { categories } = await Api.get('/api/categories');
            this.categorias = categories;
            this.renderizarFiltros();
        } catch (error) {
            console.error('Error cargando categorías:', error);
        }
    },

    // Calcular qué productos mostrar según el filtro y la búsqueda actuales
    productosFiltrados() {
        return this.productos.filter(p => {
            // ¿El producto coincide con la categoría seleccionada?
            const coincideCategoria =
                this.filtroActual === 'todos' ||
                p.category_id === this.filtroActual;

            // ¿El nombre del producto contiene el texto buscado?
            const coincideBusqueda =
                !this.busqueda ||
                p.nombre.toLowerCase().includes(this.busqueda.toLowerCase());

            // Solo incluir si cumple AMBAS condiciones
            return coincideCategoria && coincideBusqueda;
        });
    },

    // Configurar los botones de filtro por categoría (Event Delegation)
    setupFiltros() {
        document.getElementById('menuFilters').addEventListener('click', (e) => {
            if (e.target.classList.contains('filter-btn')) {
                // Actualizar el filtro activo
                this.filtroActual = e.target.dataset.category || 'todos';

                // Actualizar la apariencia de los botones
                document.querySelectorAll('.filter-btn').forEach(btn =>
                    btn.classList.remove('active')
                );
                e.target.classList.add('active');

                // Volver a renderizar con el nuevo filtro
                this.renderizarProductos(this.productosFiltrados());
            }
        });
    },

    // Configurar la búsqueda en tiempo real
    setupBusqueda() {
        const inputBusqueda = document.getElementById('searchInput');
        if (!inputBusqueda) return;

        inputBusqueda.addEventListener('input', (e) => {
            this.busqueda = e.target.value;
            this.renderizarProductos(this.productosFiltrados());
        });
    },

    // Renderizar el grid de productos en el DOM
    renderizarProductos(productos) {
        const grid = document.getElementById('productGrid');

        if (productos.length === 0) {
            grid.innerHTML = '<p class="empty">No se encontraron productos</p>';
            return;
        }

        // map() crea un array de strings HTML, join('') los concatena en uno solo
        grid.innerHTML = productos.map(p => `
            <div class="product-card">
                <div class="product-image-container">
                    <img src="${p.imagen_url || ''}" alt="${escHtml(p.nombre)}"
                         loading="lazy" onerror="this.src='img/placeholder.jpg'">
                </div>
                <div class="product-info">
                    <h3 class="product-name">${escHtml(p.nombre)}</h3>
                    <p class="product-desc">${escHtml(p.descripcion || '')}</p>
                    <div class="product-footer">
                        <span class="product-price">Bs. ${p.precio.toFixed(2)}</span>
                        <button class="btn-add-cart"
                            data-id="${p.id}"
                            data-nombre="${escHtml(p.nombre)}"
                            data-precio="${p.precio}">
                            + Agregar
                        </button>
                    </div>
                </div>
            </div>
        `).join('');

        // Event delegation para los botones de agregar
        // (si ya tiene listener del render anterior, no pasa nada: se agrega otro,
        // en producción se debería gestionar mejor con removeEventListener)
        grid.addEventListener('click', (e) => {
            if (e.target.classList.contains('btn-add-cart')) {
                Cart.addItem({
                    id: e.target.dataset.id,
                    nombre: e.target.dataset.nombre,
                    precio: parseFloat(e.target.dataset.precio)
                });
            }
        });
    }
};

Conceptos clave en menu.js

Promise.all([]) – Carga en paralelo

Promise.all() recibe un array de Promises y espera a que todas terminen. Si las lanzáramos secuencialmente (await cargarCategorias(); await cargarProductos();), esperaríamos que una termine para empezar la otra. Con Promise.all, ambas van al mismo tiempo y esperamos al que tarde más. Esto puede reducir el tiempo de carga a la mitad.

Estado local del módulo

Las propiedades productos, categorias, filtroActual y busqueda son el "estado" del módulo: datos que necesita recordar mientras el usuario interactúa con la página. Cuando el usuario cambia un filtro, solo actualizamos filtroActual y re-renderizamos, sin volver a pedir datos al servidor.

Búsqueda con .toLowerCase()

Al comparar el texto buscado con el nombre del producto, convertimos ambos a minúsculas. Así, buscar "café" también encontrará "Café", "CAFÉ" o "cafÉ". Siempre normaliza el texto antes de compararlo.

El atributo onerror en imágenes

onerror="this.src='img/placeholder.jpg'" es un fallback: si la imagen del producto no carga (URL rota, archivo eliminado), se muestra automáticamente una imagen de reemplazo. Es una práctica profesional para evitar iconos de imagen rota en la interfaz.

Resumen de la Guía 08

  • localStorage guarda datos como strings que persisten entre sesiones. Usa JSON.stringify() para guardar objetos y JSON.parse() para leerlos. Siempre protege el parse con try/catch.
  • El módulo Cart sigue el patrón "obtener → modificar → guardar" para mantener el estado del carrito consistente entre el objeto en memoria y el localStorage.
  • Una Promise representa un valor futuro. Puede estar pendiente, resuelta o rechazada. Se consume con .then()/.catch() o con async/await.
  • fetch() hace peticiones HTTP y devuelve una Promise. Siempre verifica response.ok antes de procesar la respuesta.
  • async/await hace el código asíncrono más legible al eliminar el encadenamiento de .then(). Usa try/catch/finally para manejo de errores.
  • El módulo Api centraliza todas las peticiones HTTP, agrega automáticamente el header Authorization: Bearer token y lanza errores descriptivos cuando el servidor responde con códigos de error.
  • El orden de carga de los archivos <script> en el HTML es crítico: las dependencias siempre deben ir primero. config.js es el primero; app.js es el último.
  • Promise.all([]) ejecuta múltiples Promises en paralelo y espera a que todas terminen, reduciendo el tiempo de carga total.
  • El estado local de un módulo (productos, filtroActual) evita peticiones repetidas al servidor: los datos se cargan una vez y se filtran en memoria.
  • Siempre usa escHtml() al insertar datos del servidor en el DOM para prevenir ataques XSS.
