1. Formularios en HTML – ¿Qué son y para qué sirven?

Los formularios son la forma principal en que los usuarios envían información a una aplicación web. Sin formularios, una página web sería solo lectura: no podrías registrarte, iniciar sesión, hacer un pedido ni enviar un mensaje.

Analogía: Un formulario HTML es exactamente igual a un formulario de papel, pero digital. Cuando vas a un banco, te dan una hoja con campos en blanco: "Nombre:", "CI:", "Monto:". Tú llenas esos campos y se lo entregas al empleado. En la web, el formulario HTML es esa hoja, los <input> son los campos en blanco, y el botón "Enviar" reemplaza al empleado que recibe el papel.

Estructura básica de un formulario

Todo formulario empieza con la etiqueta <form> y termina con </form>. Dentro van los campos y el botón de envío:

<form id="miFormulario" action="/ruta" method="POST">
    <label for="nombre">Tu nombre:</label>
    <input type="text" id="nombre" name="nombre" placeholder="Escribe aquí" required>

    <button type="submit">Enviar</button>
</form>

¿Qué significa cada parte?

  • <form> — El contenedor principal. Todo lo que está adentro pertenece al mismo formulario.
  • action="/ruta" — La URL a donde se envían los datos cuando el usuario hace clic en "Enviar". En Aroma no usamos action porque JavaScript intercepta el envío.
  • method="POST" — El método HTTP que se usa para enviar los datos. GET los pone en la URL (visible), POST los envía ocultos en el cuerpo de la petición (para datos sensibles como contraseñas).
  • <label for="nombre"> — La etiqueta visible que describe el campo. El atributo for debe coincidir con el id del <input> correspondiente.
  • <input> — El campo donde el usuario escribe o selecciona. Es un elemento vacío (sin etiqueta de cierre).
  • placeholder — Texto gris de ejemplo que desaparece cuando el usuario empieza a escribir.
  • required — Atributo booleano: el campo es obligatorio. El formulario no se enviará si está vacío.
  • <button type="submit"> — El botón que envía el formulario. type="submit" es el comportamiento por defecto.
Importante – En el proyecto Aroma: Usamos method sin action porque JavaScript intercepta el evento submit del formulario. En lugar de que el navegador recargue la página y envíe los datos a un servidor, nuestro código JS toma el control, procesa los datos y actualiza la interfaz dinámicamente. Esto es lo que hace que Aroma sea una SPA (Single Page Application).

2. Tipos de Input – La variedad de campos disponibles

El atributo type del elemento <input> determina qué tipo de dato se puede ingresar y cómo aparece visualmente en el navegador. Cada tipo tiene un comportamiento especial que el navegador gestiona automáticamente.

Tipo Para qué se usa Ejemplo en Aroma
type="text" Texto libre de una línea Nombre del cliente
type="email" Correo electrónico (valida formato automáticamente) Login de usuario
type="password" Contraseña (muestra puntos en lugar de letras) Contraseña de acceso
type="tel" Número de teléfono (teclado numérico en móviles) Teléfono para delivery
type="number" Valores numéricos con flechas para incrementar Cantidad de productos
type="radio" Elegir UNA opción de un grupo Método de pago (QR o efectivo)
type="checkbox" Marcar o desmarcar una opción independiente Aceptar términos y condiciones
type="file" Subir archivos desde el dispositivo Comprobante de pago
type="url" URL web (valida formato automáticamente) URL de imagen del producto
type="submit" Botón que envía el formulario Confirmar pedido

Ejemplos de código por tipo

<!-- Texto libre -->
<input type="text" id="nombre" name="nombre" placeholder="Tu nombre completo">

<!-- Email: el navegador valida que tenga @ y dominio -->
<input type="email" id="email" name="email" placeholder="tu@correo.com">

<!-- Contraseña: muestra ••••• en lugar de las letras -->
<input type="password" id="password" name="password" placeholder="Tu contraseña">

<!-- Teléfono: en móvil abre teclado numérico -->
<input type="tel" id="telefono" name="telefono" placeholder="70000000">

<!-- Número: acepta solo dígitos, con flechas arriba/abajo -->
<input type="number" id="cantidad" name="cantidad" min="1" max="99" value="1">

<!-- Radio: solo se puede elegir UNO del mismo name -->
<input type="radio" id="pagoQr" name="pago" value="qr">
<label for="pagoQr">Pago con QR</label>

<input type="radio" id="pagoEfectivo" name="pago" value="efectivo">
<label for="pagoEfectivo">Efectivo</label>

<!-- Checkbox: independiente, puede estar marcado o no -->
<input type="checkbox" id="terminos" name="terminos" value="aceptado">
<label for="terminos">Acepto los términos y condiciones</label>

<!-- File: abre el explorador de archivos -->
<input type="file" id="comprobante" name="comprobante" accept="image/*">

<!-- URL: valida que empiece con http:// o https:// -->
<input type="url" id="imagen" name="imagen" placeholder="https://...">
Tip: Los inputs de tipo radio que tienen el mismo atributo name forman un grupo automáticamente. Solo se puede seleccionar uno del grupo. En cambio, los checkbox son independientes entre sí aunque tengan el mismo name.

3. Validaciones HTML Nativas – Sin JavaScript

Una de las características más poderosas de HTML5 es que permite validar formularios directamente en el navegador, sin escribir una sola línea de JavaScript. Esto se llama validación del lado del cliente.

Atributos de validación disponibles

  • required — El campo es obligatorio. El formulario no se enviará si está vacío.
  • minlength="2" — Mínimo de caracteres que debe tener el texto.
  • maxlength="50" — Máximo de caracteres permitidos (el usuario no puede escribir más).
  • min="1" — Valor numérico mínimo aceptado (para type="number").
  • max="100" — Valor numérico máximo aceptado.
  • pattern="[A-Za-z]+" — Expresión regular que el valor debe cumplir.
  • type="email" — Valida automáticamente que tenga formato de correo.
  • type="url" — Valida automáticamente que tenga formato de URL.

Ejemplo práctico con validaciones

<form id="formularioContacto">

    <!-- Mínimo 2 letras, máximo 50 -->
    <input type="text" name="nombre"
           required
           minlength="2"
           maxlength="50"
           placeholder="Tu nombre completo">

    <!-- El navegador valida el formato de email automáticamente -->
    <input type="email" name="email"
           required
           placeholder="tu@correo.com">

    <!-- Pattern: exactamente 8 dígitos del 0 al 9 -->
    <input type="tel" name="telefono"
           pattern="[0-9]{8}"
           placeholder="12345678">

    <!-- Contraseña con mínimo de seguridad -->
    <input type="password" name="password"
           required
           minlength="6"
           maxlength="100"
           placeholder="Mínimo 6 caracteres">

    <!-- Número entre 1 y 50 -->
    <input type="number" name="cantidad"
           min="1"
           max="50"
           value="1">

    <button type="submit">Enviar</button>
</form>
Atención: Las validaciones HTML son útiles para dar retroalimentación rápida al usuario, pero no son suficientes por sí solas para seguridad. Un usuario malintencionado puede desactivarlas desde las herramientas del navegador. Siempre debes validar también en el servidor (backend). En Aroma, hacemos ambas cosas.

4. Select, Textarea y Fieldset – Más elementos de formulario

Además de <input>, los formularios HTML tienen otros elementos igualmente importantes para recopilar diferentes tipos de información.

El elemento <select> (lista desplegable)

Se usa cuando el usuario debe elegir una opción de una lista predefinida. Cada opción se define con <option>:

<!-- Lista desplegable de categorías -->
<select name="categoria" id="categoria">
    <option value="">Selecciona una categoría</option>
    <option value="cafes">Cafés</option>
    <option value="jugos">Jugos</option>
    <option value="pasteles">Pasteles</option>
    <option value="snacks">Snacks</option>
</select>
Tip: El primer <option> con value="" actúa como texto de instrucción. Si agregas required al <select>, el formulario no se enviará hasta que el usuario elija una opción real (con valor no vacío).

El elemento <textarea> (área de texto)

Para textos largos de múltiples líneas, como comentarios, descripciones o notas:

<!-- Área de texto para notas del pedido -->
<textarea name="comentarios"
          id="comentarios"
          rows="4"
          placeholder="Alguna nota especial para tu pedido..."></textarea>

El elemento <fieldset> y <legend> (agrupación)

Agrupa campos relacionados y les pone un título. Muy útil para botones de radio o cuando hay varias secciones en un formulario largo:

<!-- Agrupando opciones de pago -->
<fieldset>
    <legend>Método de pago</legend>

    <label>
        <input type="radio" name="pago" value="qr">
        Pago con QR
    </label>

    <label>
        <input type="radio" name="pago" value="efectivo">
        Efectivo
    </label>
</fieldset>

<!-- Agrupando datos personales -->
<fieldset>
    <legend>Datos de entrega</legend>

    <label for="nombreCliente">Nombre completo</label>
    <input type="text" id="nombreCliente" name="nombre" required>

    <label for="telefonoCliente">Teléfono</label>
    <input type="tel" id="telefonoCliente" name="telefono" required>
</fieldset>
Analogía: El <fieldset> es como dibujar un recuadro alrededor de un grupo de preguntas en un formulario de papel. El <legend> es el título que va en la parte superior de ese recuadro, como "Información personal" o "Datos de pago".

5. El atributo data-* – Atributos personalizados

HTML permite que nosotros inventemos nuestros propios atributos para guardar información extra en cualquier elemento. Estos atributos deben empezar con data- seguido del nombre que nosotros elijamos.

¿Para qué sirven?

Los atributos data-* son perfectos para guardar información que necesita JavaScript pero que no cambia la apariencia visual del elemento. En Aroma los usamos extensamente:

<!-- data-tab identifica qué pestaña debe mostrar cada enlace de navegación -->
<a href="#" class="nav-link" data-tab="menu">Menú</a>
<a href="#" class="nav-link" data-tab="cart">Carrito</a>
<a href="#" class="nav-link" data-tab="login">Mi Cuenta</a>

<!-- data-id, data-nombre y data-precio guardan info del producto en el botón -->
<button class="btn-agregar"
        data-id="123"
        data-nombre="Café Latte"
        data-precio="18.00">
    Agregar al carrito
</button>

<!-- data-categoria filtra los productos del menú -->
<button class="filter-btn" data-categoria="cafes">Cafés</button>
<button class="filter-btn" data-categoria="jugos">Jugos</button>

¿Cómo lee JavaScript estos atributos?

JavaScript accede a los atributos data-* a través de la propiedad dataset del elemento. El nombre del atributo (sin el prefijo data-) se convierte en una propiedad del objeto dataset:

// Si el HTML tiene: data-tab="menu"
const enlace = document.querySelector('.nav-link');
console.log(enlace.dataset.tab);       // "menu"

// Si el HTML tiene: data-id="123" data-nombre="Café Latte" data-precio="18.00"
const boton = document.querySelector('.btn-agregar');
console.log(boton.dataset.id);         // "123"
console.log(boton.dataset.nombre);     // "Café Latte"
console.log(boton.dataset.precio);     // "18.00"

// Así se usa en el proyecto Aroma para agregar al carrito:
boton.addEventListener('click', function() {
    const producto = {
        id: this.dataset.id,
        nombre: this.dataset.nombre,
        precio: parseFloat(this.dataset.precio)
    };
    agregarAlCarrito(producto);
});
Tip: Los nombres con guión en HTML (data-nombre-largo) se convierten en camelCase en JavaScript (element.dataset.nombreLargo). Esto es conversión automática del navegador.

6. Estructura completa del HTML de Aroma

Cafetería Aroma es una SPA (Single Page Application). Esto significa que toda la aplicación vive en un solo archivo HTML. En lugar de navegar a páginas diferentes, se muestran y ocultan secciones llamadas tabs. JavaScript controla cuál tab está visible en cada momento.

Arquitectura de tabs del proyecto Aroma

  • #tab-home — Página de inicio con hero y destacados
  • #tab-menu — Catálogo completo de productos
  • #tab-cart — Carrito de compras y checkout
  • #tab-orders — Historial de pedidos del usuario
  • #tab-login — Login y registro de cuenta
  • #tab-admin — Panel de administración (solo para admins)

Tab Login – Formularios de acceso y registro

El tab de login contiene DOS formularios: uno para iniciar sesión y otro para crear una cuenta nueva. Solo uno es visible a la vez, y JavaScript controla cuál mostrar según el botón que presione el usuario:

<section id="tab-login" class="tab-content">
    <div class="auth-container">
        <div class="auth-card">

            <!-- Pestañas internas: Login / Registro -->
            <div class="auth-tabs">
                <button class="auth-tab-btn active" data-form="login">
                    Iniciar Sesión
                </button>
                <button class="auth-tab-btn" data-form="register">
                    Registrarse
                </button>
            </div>

            <!-- Formulario de Login -->
            <form id="loginForm" class="auth-form">
                <h2>Bienvenido de vuelta</h2>

                <div class="form-group">
                    <label for="loginEmail">Correo electrónico</label>
                    <input type="email"
                           id="loginEmail"
                           placeholder="tu@correo.com"
                           required>
                </div>

                <div class="form-group">
                    <label for="loginPassword">Contraseña</label>
                    <input type="password"
                           id="loginPassword"
                           placeholder="Tu contraseña"
                           required>
                </div>

                <button type="submit" class="btn-primary">Iniciar Sesión</button>
                <div id="loginError" class="form-error"></div>
            </form>

            <!-- Formulario de Registro (oculto por defecto) -->
            <form id="registerForm" class="auth-form" style="display:none">
                <h2>Crear cuenta</h2>

                <div class="form-group">
                    <label for="registerName">Nombre completo</label>
                    <input type="text"
                           id="registerName"
                           placeholder="Tu nombre"
                           required>
                </div>

                <div class="form-group">
                    <label for="registerEmail">Correo electrónico</label>
                    <input type="email"
                           id="registerEmail"
                           placeholder="tu@correo.com"
                           required>
                </div>

                <div class="form-group">
                    <label for="registerPassword">Contraseña</label>
                    <input type="password"
                           id="registerPassword"
                           placeholder="Mínimo 6 caracteres"
                           required
                           minlength="6">
                </div>

                <button type="submit" class="btn-primary">Crear Cuenta</button>
                <div id="registerError" class="form-error"></div>
            </form>

        </div>
    </div>
</section>

Tab Checkout – Carrito y formulario de entrega

El tab del carrito tiene dos partes: la lista de productos agregados (generada dinámicamente por JavaScript) y el formulario de datos de entrega:

<section id="tab-cart" class="tab-content">
    <div class="cart-container">

        <div class="cart-items-section">

                <!-- El resumen del total se genera con JavaScript -->
            </div>

            <!-- Formulario de entrega (visible solo si hay items en el carrito) -->
            <form id="checkoutForm" class="checkout-form" style="display:none">
                <h3>Datos de Entrega</h3>

                <div class="form-group">
                    <label for="customerName">Nombre completo *</label>
                    <input type="text"
                           id="customerName"
                           placeholder="Tu nombre completo"
                           required>
                </div>

                <div class="form-group">
                    <label for="customerPhone">Teléfono *</label>
                    <input type="tel"
                           id="customerPhone"
                           placeholder="70000000"
                           required>
                </div>

                <div class="form-group">
                    <label for="deliveryAddress">Dirección de entrega *</label>
                    <input type="text"
                           id="deliveryAddress"
                           placeholder="Ej: Av. Heroínas #123"
                           required>
                </div>

                <div class="form-group">
                    <label>Método de pago *</label>
                    <div class="payment-options">
                        <label class="payment-option">
                            <input type="radio" name="paymentMethod" value="qr" required>
                            <span>📱 QR Bancario</span>
                        </label>
                        <label class="payment-option">
                            <input type="radio" name="paymentMethod" value="cash">
                            <span>💵 Efectivo</span>
                        </label>
                    </div>
                </div>

                <!-- Esta sección solo aparece si el usuario elige QR -->
                <div class="form-group" id="qrSection" style="display:none">
                    <label>Comprobante de pago (opcional)</label>
                    <input type="file" id="paymentProof" accept="image/*">
                </div>

                <div class="form-group">
                    <label for="orderNotes">Notas del pedido</label>
                    <textarea id="orderNotes"
                              rows="3"

                    ✅ Confirmar Pedido
                </button>
            </form>

        </div>
    </div>
</section>
Tip – ¿Por qué style="display:none"? Los formularios checkoutForm y qrSection empiezan ocultos con style="display:none". JavaScript los mostrará en el momento adecuado: el formulario de checkout cuando haya items en el carrito, y la sección QR cuando el usuario elija ese método de pago. Esto es una técnica muy común en SPAs.

7. Formularios en el Panel de Administración

El panel de administración de Aroma permite al dueño de la cafetería gestionar el catálogo de productos. El formulario de productos usa varios tipos de input que hemos aprendido, combinados en un diseño de dos columnas:

<form id="productForm">

    <!-- Fila de dos columnas: nombre y categoría -->
    <div class="form-row">
        <div class="form-group">
            <label for="productName">Nombre del producto *</label>
            <input type="text"
                   id="productName"
                   placeholder="Ej: Café Latte"
                   required>
        </div>
        <div class="form-group">
            <label for="productCategory">Categoría *</label>
            <select id="productCategory" required>
                <option value="">Selecciona categoría</option>
                <!-- Las opciones se cargan dinámicamente con JavaScript -->
            </select>
        </div>
    </div>

    <!-- Descripción del producto -->
    <div class="form-group">
        <label for="productDescription">Descripción</label>
        <textarea id="productDescription"
                  rows="3"
                  placeholder="Descripción del producto..."></textarea>
    </div>

    <!-- Fila de dos columnas: precio y costo -->
    <div class="form-row">
        <div class="form-group">
            <label for="productPrice">Precio de venta (Bs) *</label>
            <input type="number"
                   id="productPrice"
                   min="0"
                   step="0.50"
                   required>
        </div>
        <div class="form-group">
            <label for="productCost">Costo (Bs)</label>
            <input type="number"
                   id="productCost"
                   min="0"
                   step="0.50">
        </div>
    </div>

    <!-- URL de imagen -->
    <div class="form-group">
        <label for="productImage">URL de imagen</label>
        <input type="url"
               id="productImage"
               placeholder="https://...">
    </div>

    <!-- Checkbox: producto activo -->
    <div class="form-group">
        <label>
            <input type="checkbox" id="productActive" checked>
            Producto activo (visible en el menú)
        </label>
    </div>

    <!-- Botones de acción -->
    <div class="form-actions">
        <button type="button" id="btnCancelProduct" class="btn-secondary">
            Cancelar
        </button>
        <button type="submit" class="btn-primary">
            Guardar Producto
        </button>
    </div>

</form>

Puntos importantes de este formulario

  • step="0.50" en los inputs de precio: permite valores decimales de 0.50 en 0.50 (0.00, 0.50, 1.00, 1.50…).
  • checked en el checkbox: el atributo sin valor activa el checkbox por defecto (los productos nuevos están activos por defecto).
  • type="button" en el botón Cancelar: muy importante, sin esto el botón actuaría como submit y enviaría el formulario accidentalmente al hacer clic en Cancelar.
  • El <select> de categorías está vacío en el HTML; JavaScript lo llenará con las categorías que existan en la base de datos.

8. Accesibilidad en Formularios

La accesibilidad (accessibility o a11y) es la práctica de diseñar páginas web que puedan ser usadas por todas las personas, incluyendo quienes tienen discapacidades visuales, motoras o cognitivas. Los formularios son especialmente importantes en este aspecto.

La relación for/id es esencial

El atributo for del <label> debe coincidir exactamente con el id del <input> al que describe. Esto tiene dos beneficios importantes:

  1. Usabilidad: Al hacer clic en el texto del label, el campo de input recibe el foco automáticamente. Muy útil en dispositivos táctiles y para usuarios con poca precisión.
  2. Lectores de pantalla: Los programas que leen la pantalla en voz alta (como NVDA o VoiceOver) anuncian el label cuando el usuario llega al input. Sin esta conexión, solo escucharían "campo de texto" sin saber para qué sirve.
<!-- CORRECTO: label conectado al input con for/id -->
<label for="emailUsuario">Correo electrónico</label>
<input type="email" id="emailUsuario" name="email">

<!-- INCORRECTO: label sin conectar (el lector de pantalla no sabe qué describe) -->
<label>Correo electrónico</label>
<input type="email" name="email">

Atributos ARIA básicos

ARIA (Accessible Rich Internet Applications) son atributos adicionales que dan más información a los lectores de pantalla:

<!-- aria-label: descripción cuando no hay label visible -->
<input type="search"
       aria-label="Buscar productos en el menú"
       placeholder="Buscar...">

<!-- aria-describedby: conecta el input con un texto de ayuda -->
<input type="password"
       id="nuevaPassword"
       aria-describedby="passwordAyuda"
       required>
<small id="passwordAyuda">La contraseña debe tener al menos 6 caracteres.</small>

<!-- aria-required: indica campo obligatorio para lectores de pantalla -->
<input type="text" aria-required="true">

<!-- aria-live: anuncia cambios dinámicos (como mensajes de error) -->
<div id="loginError" aria-live="polite" class="form-error"></div>
Importante: Escribir código accesible no es solo un detalle técnico, es una responsabilidad ética. Aproximadamente el 15% de la población mundial tiene algún tipo de discapacidad. Un formulario inaccesible puede excluir a miles de personas de usar tu aplicación.

Formulario de contacto con los siguientes campos y sus validaciones:

  • Email (tipo email, obligatorio)
  • Teléfono (tipo tel, opcional, patrón de 8 dígitos)
  • Asunto: desplegable con opciones "Consulta general", "Reclamo", "Sugerencia", "Otro"
  • Mensaje (textarea, obligatorio, mínimo 10 caracteres)
  • Checkbox: "Acepto que me contacten por email"
  • Botón de envío
Recuerda usar correctamente los atributos for/id en todos los campos.
4
Bonus: Agrega un atributo data-seccion="contacto" a la sección principal y data-tipo con el tipo de cada campo de información (ej: data-tipo="telefono"). Esto sirve para que JavaScript pueda identificar y procesar estos elementos.

Resumen de la Guía 02

  • Un formulario HTML recopila datos del usuario y los envía a un servidor (o a JavaScript en una SPA como Aroma).
  • La estructura básica es: <form> + <label> + <input> + <button type="submit">.
  • Existen muchos tipos de input: text, email, password, tel, number, radio, checkbox, file, url.
  • Las validaciones nativas (required, minlength, pattern, etc.) funcionan sin JavaScript.
  • Otros elementos de formulario: <select>, <textarea>, <fieldset> y <legend>.
  • Los atributos data-* permiten guardar información extra en cualquier elemento HTML para que JavaScript la use.
  • Aroma es una SPA: todas las secciones están en un archivo HTML y se muestran/ocultan con JavaScript.
  • La accesibilidad en formularios se logra principalmente conectando <label for=""> con <input id=""> y usando atributos ARIA.

En la próxima guía aprenderemos CSS para transformar este HTML en la interfaz visual de Aroma.
