1. CORS — Por qué el navegador bloquea las peticiones

Analogía del guardia de seguridad: Imagina que el servidor backend es un edificio con guardia en la entrada. Por defecto, el guardia solo deja entrar a personas del mismo edificio. Si alguien viene de otro edificio (otro dominio u origen), el guardia lo bloquea aunque tenga buenas intenciones. CORS es el procedimiento que le dice al guardia: "está bien dejar pasar a las personas de estos otros edificios específicos".

CORS (Cross-Origin Resource Sharing) es una política de seguridad del navegador. Cuando el frontend en localhost:5500 hace una petición fetch a localhost:3000, el navegador detecta que son "orígenes" distintos (diferente puerto = diferente origen) y bloquea la petición a menos que el servidor lo permita explícitamente.

El error CORS en acción

Si el backend no tiene CORS configurado correctamente, verás este error en la consola del navegador:

Access to fetch at 'http://localhost:3000/api/products' from origin 'http://localhost:5500' has been blocked by CORS policy: No 'Access-Control-Allow-Origin' header is present on the requested resource.

La solución: configurar CORS en el backend

// backend/src/index.js
import cors from 'cors';

app.use(cors({
    origin: process.env.FRONTEND_URL,  // Solo permite peticiones de esta URL
    credentials: true                  // Necesario si usas cookies o Authorization headers
}));

// Para desarrollo, también puedes temporalmente permitir todos los orígenes:
// app.use(cors()); // ← Solo en desarrollo, NUNCA en producción
Importante: CORS es una política que aplica el navegador, no el servidor. Herramientas como Thunder Client o Postman no tienen esta restricción, por eso las peticiones funcionan ahí aunque CORS falle en el navegador. Esto puede llevar a confusión: "en Thunder Client funciona pero en la web no" — la respuesta casi siempre es CORS.

2. El módulo Api.js — Centralizando las peticiones

En lugar de escribir fetch() en cada parte del frontend, creamos un módulo centralizado que maneja automáticamente el token de autenticación, la URL base, y los errores.

// frontend/js/api.js
const Api = {

    // Obtener la URL base del backend desde la configuración
    get baseUrl() {
        return window.CONFIG?.API_BASE_URL || 'http://localhost:3000';
    },

    // Método genérico para hacer peticiones
    async request(method, endpoint, body = null) {
        const url = `${this.baseUrl}${endpoint}`;

        const headers = {
            'Content-Type': 'application/json'
        };

        // Agregar token de autenticación si el usuario está logueado
        const session = await Auth.getSession();
        if (session?.access_token) {
            headers['Authorization'] = `Bearer ${session.access_token}`;
        }

        const options = { method, headers };
        if (body) {
            options.body = JSON.stringify(body);
        }

        const response = await fetch(url, options);

        // Leer el cuerpo de la respuesta como JSON
        const data = await response.json();

        // Si el servidor devolvió un error, lanzarlo como excepción
        if (!response.ok) {
            throw new Error(data.error || `Error HTTP ${response.status}`);
        }

        return data;
    },

    // Métodos de conveniencia
    get(endpoint) {
        return this.request('GET', endpoint);
    },

    post(endpoint, body) {
        return this.request('POST', endpoint, body);
    },

    put(endpoint, body) {
        return this.request('PUT', endpoint, body);
    },

    patch(endpoint, body) {
        return this.request('PATCH', endpoint, body);
    },

    delete(endpoint) {
        return this.request('DELETE', endpoint);
    }
};
Por qué centralizar las peticiones: Si mañana necesitas cambiar la URL del backend o agregar un nuevo header a todas las peticiones (por ejemplo, un header de versión de API), solo tienes que cambiar un archivo. Sin esta centralización, tendrías que buscar y modificar cada fetch() en todo el proyecto.

3. El módulo login.js del frontend

La página de login maneja tanto el formulario de inicio de sesión como el de registro, con sus respectivas validaciones y feedback visual.

// frontend/js/pages/login.js
const LoginPage = {

    init() {
        this.setupForms();
        this.setupTabs();
    },

    setupForms() {
        // ===== FORMULARIO DE LOGIN =====
        document.getElementById('loginForm').addEventListener('submit', async (e) => {
            e.preventDefault();

            const email = document.getElementById('loginEmail').value;
            const password = document.getElementById('loginPassword').value;
            const errorDiv = document.getElementById('loginError');
            const btn = e.target.querySelector('button[type="submit"]');

            // Deshabilitar el botón para evitar doble envío
            btn.disabled = true;
            btn.textContent = 'Iniciando sesión...';
            errorDiv.textContent = '';

            try {
                await Auth.signIn(email, password);
                mostrarToast('¡Bienvenido a Aroma!', 'success');
                App.showTab('home');
                App.updateUserUI();
            } catch (error) {
                errorDiv.textContent = 'Email o contraseña incorrectos';
            } finally {
                // El bloque finally siempre se ejecuta, haya error o no
                btn.disabled = false;
                btn.textContent = 'Iniciar Sesión';
            }
        });

        // ===== FORMULARIO DE REGISTRO =====
        document.getElementById('registerForm').addEventListener('submit', async (e) => {
            e.preventDefault();

            const name = document.getElementById('registerName').value.trim();
            const email = document.getElementById('registerEmail').value.trim();
            const password = document.getElementById('registerPassword').value;
            const confirm = document.getElementById('registerConfirm').value;
            const errorDiv = document.getElementById('registerError');

            // Validación básica en el frontend
            if (password !== confirm) {
                errorDiv.textContent = 'Las contraseñas no coinciden';
                return;
            }

            if (password.length < 6) {
                errorDiv.textContent = 'La contraseña debe tener al menos 6 caracteres';
                return;
            }

            const btn = e.target.querySelector('button[type="submit"]');
            btn.disabled = true;
            btn.textContent = 'Creando cuenta...';
            errorDiv.textContent = '';

            try {
                await Auth.signUp(email, password, name);
                mostrarToast('Cuenta creada exitosamente. ¡Inicia sesión!', 'success');
                // Cambiar automáticamente al tab de login
                document.querySelector('[data-form="login"]').click();
            } catch (error) {
                errorDiv.textContent = error.message || 'Error al crear la cuenta';
            } finally {
                btn.disabled = false;
                btn.textContent = 'Crear Cuenta';
            }
        });
    },

    setupTabs() {
        document.querySelectorAll('.auth-tab-btn').forEach(btn => {
            btn.addEventListener('click', () => {
                const form = btn.dataset.form;

                // Actualizar estado activo de los tabs
                document.querySelectorAll('.auth-tab-btn').forEach(b =>
                    b.classList.remove('active')
                );
                btn.classList.add('active');

                // Mostrar/ocultar formularios
                document.getElementById('loginForm').style.display =
                    form === 'login' ? 'block' : 'none';
                document.getElementById('registerForm').style.display =
                    form === 'register' ? 'block' : 'none';
            });
        });
    }
};

El módulo Auth.js

El módulo Auth encapsula toda la comunicación con Supabase Auth desde el frontend:

// frontend/js/auth.js
const Auth = {

    async signIn(email, password) {
        const { data, error } = await supabaseClient.auth.signInWithPassword({
            email,
            password
        });
        if (error) throw error;
        return data;
    },

    async signUp(email, password, fullName) {
        const { data, error } = await supabaseClient.auth.signUp({
            email,
            password,
            options: {
                data: { full_name: fullName }
            }
        });
        if (error) throw error;
        return data;
    },

    async signOut() {
        const { error } = await supabaseClient.auth.signOut();
        if (error) throw error;
    },

    async getSession() {
        const { data: { session } } = await supabaseClient.auth.getSession();
        return session;
    },

    isLoggedIn() {
        // Verificación síncrona básica
        return !!supabaseClient.auth.currentSession;
    },

    getRole() {
        const session = supabaseClient.auth.currentSession;
        return session?.user?.user_metadata?.role || 'client';
    }
};

4. La página del carrito y el checkout

Esta es la página más crítica del flujo de usuario: muestra los productos seleccionados, permite modificar cantidades, y procesa el pedido enviándolo al backend.

// frontend/js/pages/cart-page.js
const CartPage = {

    init() {
        this.renderCart();
        this.setupCheckoutForm();
        Cart.updateBadge();
    },

    renderCart() {
        const items = Cart.getItems();
        const container = document.getElementById('cartItems');

        if (items.length === 0) {
            container.innerHTML = `
                <div class="empty-cart">
                    <p>Tu carrito está vacío</p>
                    <button onclick="App.showTab('menu')">Ver el Menú</button>
                </div>
            `;
            document.getElementById('checkoutForm').style.display = 'none';
            return;
        }

        // Renderizar items del carrito
        container.innerHTML = items.map(item => `
            <div class="cart-item" data-id="${item.id}">
                <img src="${item.imagen || ''}" alt="${escHtml(item.nombre)}">
                <div class="cart-item-info">
                    <h4>${escHtml(item.nombre)}</h4>
                    <span>Bs. ${item.precio.toFixed(2)} c/u</span>
                </div>
                <div class="cart-item-controls">
                    <button class="btn-qty" data-action="decrease" data-id="${item.id}">−</button>
                    <span class="qty">${item.cantidad}</span>
                    <button class="btn-qty" data-action="increase" data-id="${item.id}">+</button>
                </div>
                <span class="cart-item-subtotal">
                    Bs. ${(item.precio * item.cantidad).toFixed(2)}
                </span>
                <button class="btn-remove" data-id="${item.id}">Eliminar</button>
            </div>
        `).join('');

        // Mostrar total
        document.getElementById('cartSummary').innerHTML = `
            <div class="cart-total">
                <span>Total:</span>
                <strong>Bs. ${Cart.getTotal().toFixed(2)}</strong>
            </div>
        `;

        // Solo mostrar el formulario si el usuario está autenticado
        document.getElementById('checkoutForm').style.display =
            Auth.isLoggedIn() ? 'block' : 'none';

        if (!Auth.isLoggedIn()) {
            document.getElementById('cartSummary').innerHTML += `
                <p class="login-required">
                    <a href="#" onclick="App.showTab('login')">Inicia sesión</a>
                    para hacer tu pedido
                </p>
            `;
        }

        // Usar event delegation para los controles del carrito
        container.addEventListener('click', (e) => {
            const id = e.target.dataset.id;
            const action = e.target.dataset.action;

            if (e.target.classList.contains('btn-remove')) {
                Cart.removeItem(id);
                this.renderCart();
            } else if (action === 'increase') {
                const item = Cart.getItems().find(i => i.id === id);
                Cart.updateQuantity(id, item.cantidad + 1);
                this.renderCart();
            } else if (action === 'decrease') {
                const item = Cart.getItems().find(i => i.id === id);
                if (item.cantidad <= 1) {
                    Cart.removeItem(id);
                } else {
                    Cart.updateQuantity(id, item.cantidad - 1);
                }
                this.renderCart();
            }
        });
    },

    setupCheckoutForm() {
        document.getElementById('checkoutForm').addEventListener('submit', async (e) => {
            e.preventDefault();

            const items = Cart.getItems();
            if (items.length === 0) {
                mostrarToast('El carrito está vacío', 'warning');
                return;
            }

            const btn = e.target.querySelector('button[type="submit"]');
            btn.disabled = true;
            btn.textContent = 'Procesando pedido...';

            try {
                // Enviar el pedido al backend
                const respuesta = await Api.post('/api/orders', {
                    customer_name: document.getElementById('customerName').value,
                    customer_phone: document.getElementById('customerPhone').value,
                    delivery_address: document.getElementById('deliveryAddress').value,
                    payment_method: document.querySelector('[name="paymentMethod"]:checked').value,
                    notes: document.getElementById('orderNotes').value,
                    // Solo enviamos product_id y quantity, NO el precio
                    items: items.map(item => ({
                        product_id: item.id,
                        quantity: item.cantidad
                    }))
                });

                // Limpiar el carrito y mostrar confirmación
                Cart.clear();
                mostrarToast(`Pedido ${respuesta.order.code} confirmado`, 'success', 5000);
                this.mostrarConfirmacion(respuesta.order);

            } catch (error) {
                mostrarToast(error.message || 'Error al crear el pedido', 'error');
            } finally {
                btn.disabled = false;
                btn.textContent = 'Confirmar Pedido';
            }
        });
    },

    mostrarConfirmacion(order) {
        document.getElementById('tab-cart').innerHTML = `
            <div class="order-success">
                <div class="success-icon" style="font-size:3em;margin-bottom:16px">✅</div>
                <h2>Pedido Confirmado</h2>
                <p class="order-code" style="font-size:1.5em;color:#6F4E37;font-weight:bold">
                    ${order.code}
                </p>
                <p>Recibirás tu pedido en aproximadamente 30-45 minutos</p>
                <p>Total: <strong>Bs. ${order.total.toFixed(2)}</strong></p>
                <button onclick="App.showTab('home')" style="margin-top:16px">
                    Volver al Inicio
                </button>
            </div>
        `;
    }
};
Event delegation: En lugar de agregar un listener a cada botón de "aumentar", "disminuir" y "eliminar", usamos un único listener en el contenedor padre y verificamos qué elemento fue clickeado. Esto es más eficiente y funciona aunque el contenido se re-renderice dinámicamente.

5. El módulo App.js — El orquestador

App es el módulo central que inicializa la aplicación, maneja la navegación entre secciones y coordina el estado del usuario.

// frontend/js/app.js
const App = {
    currentTab: 'home',

    init() {
        document.addEventListener('DOMContentLoaded', async () => {
            // Verificar si hay una sesión guardada
            await this.checkSession();

            // Configurar los listeners de navegación
            this.setupNavigation();

            // Actualizar el badge del carrito
            Cart.updateBadge();

            // Mostrar el tab inicial
            this.showTab('home');

            // Inicializar los módulos de cada página
            if (typeof HomePage !== 'undefined') HomePage.init();
            if (typeof MenuPage !== 'undefined') MenuPage.init();
            if (typeof CartPage !== 'undefined') CartPage.init();
            if (typeof LoginPage !== 'undefined') LoginPage.init();
            if (typeof ContactPage !== 'undefined') ContactPage.init();
        });
    },

    async checkSession() {
        const session = await Auth.getSession();
        this.updateUserUI(session);

        if (session?.user) {
            const role = session.user.user_metadata?.role;
            // Mostrar tabs de cajero y admin según el rol
            if (role === 'cashier' || role === 'admin') {
                this.showNavItem('cashier');
            }
            if (role === 'admin') {
                this.showNavItem('admin');
            }
        }
    },

    updateUserUI(session) {
        const userGreeting = document.getElementById('userGreeting');
        const loginLink = document.getElementById('loginNavLink');

        if (session?.user) {
            const nombre = session.user.user_metadata?.full_name || session.user.email;
            if (userGreeting) {
                userGreeting.innerHTML = `
                    <span>${escHtml(nombre)}</span>
                    <button onclick="App.logout()" class="btn-logout">Salir</button>
                `;
            }
            if (loginLink) loginLink.style.display = 'none';
        } else {
            if (userGreeting) userGreeting.innerHTML = '';
            if (loginLink) loginLink.style.display = 'block';
        }
    },

    async logout() {
        await Auth.signOut();
        this.updateUserUI(null);
        this.showTab('home');
        mostrarToast('Sesión cerrada', 'info');
        // Ocultar tabs de cajero y admin
        this.hideNavItem('cashier');
        this.hideNavItem('admin');
    },

    showTab(tabName) {
        // Ocultar todos los tabs
        document.querySelectorAll('.tab-content').forEach(tab => {
            tab.style.display = 'none';
        });

        // Mostrar el tab solicitado
        const targetTab = document.getElementById(`tab-${tabName}`);
        if (targetTab) {
            targetTab.style.display = 'block';
            this.currentTab = tabName;
        }

        // Actualizar la navegación activa
        document.querySelectorAll('.nav-link').forEach(link => {
            link.classList.toggle('active', link.dataset.tab === tabName);
        });

        // Scroll al inicio
        window.scrollTo(0, 0);
    },

    setupNavigation() {
        document.querySelectorAll('[data-tab]').forEach(el => {
            el.addEventListener('click', (e) => {
                e.preventDefault();
                this.showTab(el.dataset.tab);
            });
        });
    },

    showNavItem(name) {
        const el = document.getElementById(`nav-${name}`);
        if (el) el.style.display = 'block';
    },

    hideNavItem(name) {
        const el = document.getElementById(`nav-${name}`);
        if (el) el.style.display = 'none';
    }
};

App.init();

6. Diagnóstico: Errores comunes de integración

La integración frontend-backend produce errores muy específicos. Aquí están los más frecuentes y sus soluciones:

Error Causa Solución
CORS policy blocked El backend no permite el origen del frontend Verificar FRONTEND_URL en el .env del backend y el middleware cors
401 Unauthorized El token no se está enviando o está expirado Verificar que Api.js agrega el header Authorization. Hacer login nuevamente si el token expiró
404 Cannot GET /api/... El servidor no está corriendo o la ruta está mal escrita Verificar que el backend está corriendo (npm run dev) y que la ruta existe
Failed to fetch / NetworkError El backend no está accesible desde el frontend Probar http://localhost:3000/health en el navegador. Verificar puerto
Productos no aparecen Están marcados como inactivos en la BD Verificar que activo = true en los registros de la tabla products
Total incorrecto Los precios se están guardando como strings en lugar de números Usar parseFloat() al leer precios, verificar el tipo en Supabase (numeric, no text)
req.body es undefined Falta el middleware express.json() en el backend Agregar app.use(express.json()) en index.js antes de las rutas

Herramientas de diagnóstico

Tip — Usar la consola del navegador: La pestaña "Network" (Red) de las DevTools del navegador (F12) muestra todas las peticiones HTTP, su estado, los headers enviados y recibidos, y la respuesta. Es la primera herramienta a usar cuando algo falla en la integración.

7. El módulo Cart.js — Gestión del carrito con localStorage

El carrito persiste en el navegador usando localStorage para que no se pierda al recargar la página.

// frontend/js/cart.js
const Cart = {

    getItems() {
        const items = localStorage.getItem('aroma_cart');
        return items ? JSON.parse(items) : [];
    },

    saveItems(items) {
        localStorage.setItem('aroma_cart', JSON.stringify(items));
        this.updateBadge();
    },

    addItem(product) {
        const items = this.getItems();
        const existing = items.find(i => i.id === product.id);

        if (existing) {
            // Si ya está en el carrito, aumentar la cantidad
            existing.cantidad += 1;
        } else {
            // Agregar nuevo item con cantidad 1
            items.push({ ...product, cantidad: 1 });
        }

        this.saveItems(items);
        mostrarToast(`${product.nombre} agregado al carrito`, 'success');
    },

    removeItem(productId) {
        const items = this.getItems().filter(i => i.id !== productId);
        this.saveItems(items);
    },

    updateQuantity(productId, newQty) {
        if (newQty <= 0) {
            this.removeItem(productId);
            return;
        }
        const items = this.getItems().map(i =>
            i.id === productId ? { ...i, cantidad: newQty } : i
        );
        this.saveItems(items);
    },

    getTotal() {
        return this.getItems().reduce((sum, i) => sum + (i.precio * i.cantidad), 0);
    },

    getCount() {
        return this.getItems().reduce((sum, i) => sum + i.cantidad, 0);
    },

    clear() {
        localStorage.removeItem('aroma_cart');
        this.updateBadge();
    },

    updateBadge() {
        const badge = document.getElementById('cartBadge');
        const count = this.getCount();
        if (badge) {
            badge.textContent = count;
            badge.style.display = count > 0 ? 'flex' : 'none';
        }
    }
};

Resumen de la Guía 13

  • CORS es la política del navegador que bloquea peticiones entre orígenes distintos; se configura en el backend con el middleware cors.
  • El módulo Api.js centraliza todas las peticiones y agrega automáticamente el token de autenticación.
  • El módulo LoginPage maneja login y registro con feedback visual y gestión de errores.
  • El checkout envía solo product_id y quantity al backend; los precios los calcula el servidor.
  • El módulo App orquesta la navegación, verifica la sesión al inicio y actualiza la UI según el rol.
  • El carrito persiste en localStorage y se sincroniza con la UI mediante el badge.
  • La pestaña Network de DevTools es la herramienta principal para depurar problemas de integración.
