1. ¿Qué es HTML?

HTML son las siglas de HyperText Markup Language, que en español significa "Lenguaje de Marcado de Hipertexto". Vamos a entender cada palabra:

  • HyperText = "texto con enlaces". El "hiper" significa que podemos ir más allá de un documento lineal, saltando de una página a otra mediante enlaces (links).
  • Markup = "marcado". Usamos etiquetas especiales para marcar el texto y decirle al navegador cómo debe interpretarlo.
  • Language = lenguaje. Es un conjunto de reglas y símbolos con significado propio.
Analogía del esqueleto humano:
Una página web tiene tres capas, igual que una persona:
  • HTML = El esqueleto: Da estructura y forma. Sin él no hay nada. Define que hay una cabeza, un cuerpo, dos piernas, etc.
  • CSS = La ropa y el aspecto: Da color, forma, belleza. El mismo esqueleto puede verse muy diferente con distintas "ropas".
  • JavaScript = Los músculos y el cerebro: Da movimiento, interactividad e inteligencia. Permite que la persona haga cosas.
En esta guía nos enfocamos en el esqueleto.

¿HTML es un lenguaje de programación?

Aclaración importante: HTML NO es un lenguaje de programación. Es un lenguaje de marcado. La diferencia es fundamental:
  • Los lenguajes de programación (JavaScript, Python, Java) permiten crear lógica: condiciones (if/else), bucles (for/while), funciones, variables.
  • Los lenguajes de marcado (HTML, XML) solo describen la estructura y el significado del contenido. No hay lógica.
HTML le dice al navegador: "esto es un título", "esto es un párrafo", "esto es una imagen". No calcula ni toma decisiones.

¿Cómo funciona HTML?

1
Tú escribes un archivo de texto con extensión .html usando etiquetas especiales como <h1>, <p>, <img>.
2
El navegador (Chrome, Firefox, Edge) lee ese archivo de arriba a abajo.
3
El navegador interpreta cada etiqueta y construye internamente un árbol de objetos llamado DOM (Document Object Model).
4
El navegador dibuja (renderiza) el resultado en pantalla: títulos grandes, párrafos de texto, imágenes, botones, etc.
Puedes ver el HTML de cualquier página: En Chrome, presiona Ctrl+U en cualquier página web para ver su código HTML fuente. O presiona F12 y ve a la pestaña "Elements" para verlo de forma interactiva.

2. Mi primer archivo HTML

Todo documento HTML válido tiene la misma estructura base. Memoriza esta plantilla porque la usarás en cada proyecto que hagas en tu vida:

<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Mi Página</title>
</head>
<body>
    <!-- Aquí va el contenido visible -->
    <h1>Hola Mundo</h1>
</body>
</html>

Expliquemos cada línea en detalle:

Línea por línea

<!DOCTYPE html>

Esta no es una etiqueta HTML sino una declaración que va al principio del archivo. Le dice al navegador: "Este archivo usa la versión moderna de HTML (HTML5)". Sin esto, el navegador puede interpretar el documento en modo antiguo y las cosas pueden verse raro. Siempre debe ser la primera línea.

<html lang="es">

La etiqueta <html> es la raíz del documento. Todo el contenido de la página va dentro de ella. El atributo lang="es" le dice al navegador y a los motores de búsqueda que el idioma de la página es español. Esto también ayuda a los lectores de pantalla (para personas con discapacidad visual) a pronunciar correctamente el texto.

<head>

La sección <head> contiene metadatos: información SOBRE la página que el navegador necesita, pero que el usuario generalmente no ve directamente. Aquí van el título (que aparece en la pestaña), los estilos CSS, los scripts, y configuraciones importantes.

<meta charset="UTF-8">

Le dice al navegador qué codificación de caracteres usar. UTF-8 es el estándar universal que incluye todos los caracteres: letras del español (ñ, á, é, í, ó, ú), emojis, caracteres chinos, árabes, etc. Sin esta línea, las tildes y la ñ pueden verse como símbolos extraños.

<meta name="viewport" content="width=device-width, initial-scale=1.0">

Esta línea es indispensable para el diseño responsivo (que la página se vea bien en celulares). Sin ella, los móviles hacen zoom-out para mostrar la versión "de escritorio" muy pequeña. Con ella, la página se adapta al ancho del dispositivo. En la era del móvil, esta línea es obligatoria.

<title>Mi Página</title>

El texto dentro de <title> es lo que aparece en la pestaña del navegador y en los resultados de Google. Es muy importante para el SEO (posicionamiento en buscadores). Debe ser descriptivo y único para cada página.

<body>

Todo lo que el usuario ve en la pantalla va dentro del <body>. Títulos, párrafos, imágenes, botones, formularios, todo. Es el "cuerpo" visible del documento.

<!-- Comentario -->

Los comentarios son notas para los desarrolladores. El navegador los ignora completamente y el usuario no los ve. Son muy útiles para explicar qué hace una sección del código. Úsalos generosamente: tu yo del futuro te lo agradecerá.

Prueba: Crea tu primer archivo HTML

1
Abre VS Code. Crea un archivo nuevo con Ctrl+N.
2
Guárdalo como mi-primera-pagina.html con Ctrl+S.
3
Escribe ! y presiona Tab. VS Code generará automáticamente la estructura base completa (esto es Emmet, una herramienta integrada en VS Code).
4
Dentro del <body>, escribe <h1>Hola, soy una página web</h1>
5
Haz clic derecho en el archivo en el explorador de VS Code y elige "Open with Live Server". Verás tu página en el navegador.

3. Etiquetas HTML esenciales

HTML tiene más de 100 etiquetas, pero con 15-20 puedes construir cualquier cosa. Veamos las más importantes.

1 Encabezados: h1 al h6

Los encabezados crean jerarquía visual en el texto. Son como los títulos y subtítulos de un libro.

<h1>Cafetería Aroma</h1>           <!-- El título más grande -->
<h2>Nuestro Menú</h2>              <!-- Sección principal -->
<h3>Bebidas Calientes</h3>         <!-- Subsección -->
<h4>Café Americano</h4>            <!-- Sub-subsección -->
<h5>Ingredientes</h5>              <!-- Raramente necesario -->
<h6>Nota del barista</h6>          <!-- Raramente necesario -->
Regla importante: Solo debe haber un <h1> por página. Es el título principal, el más importante. Google lo usa para entender de qué trata la página. Puedes tener muchos <h2>, <h3>, etc., pero el <h1> es único.

2 Párrafos y texto: p, strong, em, span

<p>Bienvenido a Cafetería Aroma, donde cada sorbo es una experiencia.</p>

<p>
    Nuestro café viene de <strong>granos seleccionados</strong> de las
    mejores regiones de Bolivia. Cada taza es preparada con
    <em>dedicación y pasión</em> por nuestros baristas.
</p>

<p>
    El precio especial del día es
    <span style="color: red;">Bs. 15</span>.
</p>
Etiqueta Significado Resultado visual Cuándo usarla
<p> Paragraph (párrafo) Bloque de texto con espacio arriba y abajo Para cualquier bloque de texto
<strong> Fuerte/importante Texto en negrita Texto que es semánticamente importante
<em> Emphasis (énfasis) Texto en cursiva Texto que requiere énfasis verbal
<span> Sin significado propio Texto sin cambio visual Para aplicar estilos a texto dentro de un párrafo
<br> Line break (salto de línea) Salto de línea sin nuevo párrafo Cuando necesitas un salto pero no un párrafo nuevo
No confundas <strong> con <b>: Ambas hacen negrita visualmente, pero tienen significados distintos. <strong> significa "este texto es importante" (semántica). <b> solo dice "pon esto en negrita" (visual). Siempre prefiere <strong>. Lo mismo aplica para <em> vs <i>.

3 Listas: ul, ol, li

Las listas son esenciales. En Cafetería Aroma las usamos para la barra de navegación, los ingredientes de los productos y las opciones del menú.

<!-- Lista desordenada (sin numeración): puntos/viñetas -->
<ul>
    <li>Café Americano</li>
    <li>Café Latte</li>
    <li>Cappuccino</li>
    <li>Té Verde</li>
</ul>

<!-- Lista ordenada (con numeración): pasos, rankings -->
<ol>
    <li>Elige tus productos en el menú</li>
    <li>Agrégralos al carrito</li>
    <li>Revisa tu pedido</li>
    <li>Elige cómo pagar</li>
    <li>¡Listo! Recoge tu pedido</li>
</ol>

<!-- Lista anidada: listas dentro de listas -->
<ul>
    <li>Bebidas calientes
        <ul>
            <li>Café Americano</li>
            <li>Café con leche</li>
        </ul>
    </li>
    <li>Bebidas frías
        <ul>
            <li>Frappé</li>
            <li>Limonada</li>
        </ul>
    </li>
</ul>
La navegación de un sitio web es una lista: Aunque la barra de navegación se vea horizontal y elegante gracias a CSS, en el HTML es simplemente un <ul> con <li>. Esto tiene sentido semánticamente porque es una lista de opciones.

4 Imágenes: img

Las imágenes hacen que una página cobre vida. En Aroma, cada producto del menú tiene una foto.

<!-- Imagen básica -->
<img src="img/cafe-latte.jpg" alt="Un café latte con arte en la espuma">

<!-- Imagen con dimensiones definidas -->
<img src="img/logo.png" alt="Logo de Cafetería Aroma" width="200" height="80">

<!-- Imagen de internet (URL externa) -->
<img src="https://ejemplo.com/imagen.jpg" alt="Descripción de la imagen">
El atributo alt es OBLIGATORIO: El texto alternativo (alt) es fundamental por tres razones:
  1. Accesibilidad: Los lectores de pantalla (usados por personas ciegas) leen el alt en voz alta para describir la imagen.
  2. SEO: Google no puede "ver" imágenes, pero sí puede leer el alt. Descríbelo con palabras clave.
  3. Fallback: Si la imagen no carga (mala conexión, ruta incorrecta), el usuario ve el texto del alt en su lugar.

El atributo src (source = fuente) indica la ruta de la imagen. Puede ser:

  • Ruta relativa: img/foto.jpg — busca el archivo relativo a la ubicación del HTML.
  • Ruta absoluta desde la raíz: /frontend/img/foto.jpg — desde la raíz del servidor.
  • URL completa: https://supabase.co/storage/.../foto.jpg — una imagen en otro servidor.

5 Enlaces: a

Los enlaces son lo que hace de la web una "web" (una red interconectada). La etiqueta <a> significa "anchor" (ancla).

<!-- Enlace a otra página web -->
<a href="https://www.google.com">Ir a Google</a>

<!-- Enlace que abre en una pestaña nueva -->
<a href="https://supabase.com" target="_blank" rel="noopener">
    Visitar Supabase
</a>

<!-- Enlace a otra página del mismo sitio (ruta relativa) -->
<a href="menu.html">Ver el Menú</a>

<!-- Enlace a una sección dentro de la misma página (ancla) -->
<a href="#contacto">Ir a Contacto</a>
...
<section id="contacto">...</section>

<!-- Enlace de email -->
<a href="mailto:info@cafeteria-aroma.com">Envíanos un email</a>

<!-- Enlace de teléfono (para móviles) -->
<a href="tel:+59177123456">Llamar al local</a>

<!-- Enlace que parece un botón (el estilo lo da CSS) -->
<a href="menu.html" class="btn-primary">Ver Menú</a>
Sobre target="_blank": Abre el enlace en una pestaña nueva. Siempre acompáñalo con rel="noopener noreferrer" por seguridad: evita que la página de destino pueda acceder a tu página mediante JavaScript.

6 Contenedores: div y span

Los contenedores son etiquetas que no tienen significado semántico propio: son "cajas" para agrupar y organizar otros elementos.

<!-- div: contenedor de BLOQUE (ocupa toda la fila) -->
<div class="tarjeta-producto">
    <img src="cafe.jpg" alt="Café">
    <h3>Café Americano</h3>
    <p>El clásico de siempre.</p>
    <span class="precio">Bs. 15</span>
</div>

<!-- span: contenedor EN LÍNEA (solo ocupa lo que necesita) -->
<p>
    El <span class="nombre-producto">Café Latte</span>
    cuesta <span class="precio">Bs. 25</span>.
</p>
<div> <span>
Tipo Elemento de bloque Elemento en línea
Comportamiento Ocupa toda la fila disponible, el siguiente elemento va en la línea siguiente Solo ocupa el espacio de su contenido, el siguiente elemento continúa en la misma línea
Uso típico Agrupar secciones, tarjetas, columnas Dar estilo a parte de un texto

7 Formularios: form, input, button

Los formularios permiten al usuario enviar información. En Aroma los usamos para el login, el registro y el formulario de contacto.

<form action="/login" method="POST">

    <!-- Etiqueta del campo -->
    <label for="email">Correo electrónico:</label>
    <input
        type="email"
        id="email"
        name="email"
        placeholder="tu@email.com"
        required
    >

    <label for="password">Contraseña:</label>
    <input
        type="password"
        id="password"
        name="password"
        placeholder="Tu contraseña"
        required
    >

    <button type="submit">Iniciar Sesión</button>
    <button type="button" onclick="limpiarFormulario()">Limpiar</button>

</form>

4. Atributos HTML

Los atributos son información adicional que le damos a las etiquetas. Van siempre dentro de la etiqueta de apertura, con el formato nombre="valor".

Analogía de los adjetivos:
Si las etiquetas HTML son sustantivos ("imagen", "enlace", "párrafo"), los atributos son adjetivos que los describen y modifican. No es lo mismo "una imagen" que "una imagen con fuente 'foto.jpg', ancho 300px y texto alternativo 'café caliente'". 
<!-- Anatomía de una etiqueta con atributos -->
<img src="cafe.jpg" alt="Café caliente" width="300" class="foto-producto" id="foto-principal">

Los atributos más importantes

Atributo ¿Qué hace? Ejemplo Nota
id Identificador único en toda la página id="boton-carrito" No puede repetirse. JavaScript lo usa para encontrar elementos específicos.
class Clasifica el elemento en un grupo class="tarjeta-producto" Puede repetirse en múltiples elementos. CSS y JS lo usan para aplicar estilos/comportamientos a grupos.
href URL de destino del enlace href="https://..." Solo para <a> y <link>.
src Ruta del recurso externo src="img/cafe.jpg" Para <img>, <script>, <iframe>.

Los atributos data-*: Son atributos personalizados que tú mismo inventas. El prefijo data- es obligatorio. Permiten guardar información extra en los elementos HTML que JavaScript puede leer. En Aroma usamos data-tab="menu" para saber a qué pestaña pertenece cada enlace de la navegación.

5. HTML Semántico

Una de las decisiones de diseño más importantes en HTML moderno es elegir las etiquetas correctas. Aquí es donde entra el concepto de semántica.

Analogía del lenguaje humano:
"Ese objeto redondo y rojo que se come" y "manzana" dicen lo mismo, pero la segunda es mucho más informativa y precisa. En HTML, usar <nav> en vez de <div class="nav"> le dice exactamente al navegador, a Google y a los lectores de pantalla que "esto es una barra de navegación", sin ambigüedad.

¿Por qué importa el HTML semántico?

  • Accesibilidad: Los lectores de pantalla (para personas con discapacidad visual) usan las etiquetas semánticas para navegar la página. Un usuario ciego puede saltar directamente a la <nav> o al <main>.
  • SEO (posicionamiento en buscadores): Google entiende mejor la estructura de tu página y puede indexarla de forma más precisa.
  • Legibilidad del código: Cuando lees <header>, sabes instantáneamente qué es. Con <div class="header">, tienes que confiar en que el nombre de la clase sea descriptivo.

Las etiquetas semánticas principales

<!-- Estructura semántica de una página web típica -->
<body>

    <header>
        <!-- Encabezado de la página o sección: logo, título, eslogan -->
        <h1>Cafetería Aroma</h1>
    </header>

    <nav>
        <!-- Navegación principal del sitio -->
        <ul>
            <li><a href="/">Inicio</a></li>
            <li><a href="/menu">Menú</a></li>
        </ul>
    </nav>

    <main>
        <!-- Contenido principal y único de la página -->

        <section>
            <!-- Sección temática: un grupo de contenido relacionado -->
            <h2>Nuestros Productos Destacados</h2>

            <article>
                <!-- Contenido independiente que tiene sentido por sí solo -->
                <h3>Café Americano</h3>
                <p>El clásico infalible.</p>
            </article>

        </section>

        <aside>
            <!-- Contenido secundario, relacionado pero no esencial -->
            <h3>Promoción del día</h3>
            <p>2x1 en frappés</p>
        </aside>

    </main>

    <footer>
        <!-- Pie de página: información de contacto, copyright, links -->
        <p>© 2025 Cafetería Aroma. Todos los derechos reservados.</p>
    </footer>

</body>

Comparación: sin semántica vs. con semántica

❌ Sin semántica (evitar)

<div class="header">
  <div class="logo">Aroma</div>
</div>
<div class="nav">
  <div class="nav-item">Menú</div>
</div>
<div class="contenido">
  <div class="seccion">
    <div class="articulo">...</div>
  </div>
</div>
<div class="footer">...</div>

✅ Con semántica (correcto)

<header>
  <h1>Aroma</h1>
</header>
<nav>
  <a href="/menu">Menú</a>
</nav>
<main>
  <section>
    <article>...</article>
  </section>
</main>
<footer>...</footer>
Etiqueta Significado Cuándo usarla
<header> Encabezado Logo, título de la página, o encabezado de una sección/artículo
<nav> Navegación Menú de navegación principal o secundario
<main> Contenido principal El contenido único y principal de la página. Solo uno por página.
<section> Sección temática Grupo de contenido relacionado con un tema específico
<article> Contenido independiente Contenido que tiene sentido por sí solo: una noticia, un producto, un post
<aside> Contenido lateral Publicidad, widgets, información complementaria
<footer> Pie de página Copyright, links legales, datos de contacto

6. Ejercicio – Mi primera página web completa

Crea una página web para una cafetería imaginaria

Sigue estos pasos para crear tu primera página web completa. Abre VS Code, crea un nuevo archivo llamado practica-html.html y escribe el siguiente código:

7. Aplicando en Cafetería Aroma – El HTML real del proyecto

Ahora vamos a analizar el archivo HTML real del proyecto Cafetería Aroma. Este es el frontend/index.html que ya existe en nuestro repositorio:

<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Cafetería Aroma</title>
    <link rel="stylesheet" href="css/styles.css">
</head>

Análisis detallado: cada parte explicada

¿Por qué data-tab en los links de navegación?

<a href="#" class="nav-link" data-tab="menu">Menú</a>

En lugar de apuntar a otra página (href="menu.html"), el enlace apunta a # (no va a ningún lado). El atributo data-tab="menu" es como un "mensaje secreto" para JavaScript. Cuando el usuario hace clic, JavaScript lee el valor de data-tab y sabe que debe mostrar la sección con id="tab-menu" y ocultar todas las demás. Así funciona el sistema de pestañas sin recargar la página.

¿Qué es el sistema de "tabs" o pestañas?

Aroma usa una técnica donde todas las secciones de la aplicación existen en el HTML al mismo tiempo, pero solo una se muestra a la vez. CSS oculta las secciones con display: none, y JavaScript las muestra/oculta según el enlace que el usuario clicó. Esto evita tener que cargar una nueva página cada vez que el usuario navega.

<!-- Todas las secciones existen en el DOM, pero solo una tiene la clase "active" -->
<section id="tab-home" class="tab-content active">...</section>   <!-- VISIBLE -->
<section id="tab-menu" class="tab-content">...</section>          <!-- OCULTO -->
<section id="tab-cart" class="tab-content">...</section>          <!-- OCULTO -->

¿Por qué el productGrid está vacío en el HTML?

<div id="productGrid" class="product-grid">
    <!-- Los productos se cargan con JavaScript -->
</div>

El div con id productGrid está vacío intencionalmente. Los productos no están escritos en el HTML porque vienen de la base de datos (Supabase). Cuando la aplicación carga, JavaScript hace una petición al servidor, recibe la lista de productos en formato JSON, y genera dinámicamente el HTML de cada tarjeta de producto. Esto permite que:

  • El administrador agregue o quite productos desde el panel, sin tocar el HTML.
  • Los precios se actualicen automáticamente.
  • Solo se muestren los productos disponibles.

¿Por qué los scripts están al final del body?

<!-- Al final del <body>, NO en el <head> -->
<script src="js/config.js"></script>
<script src="js/app.js"></script>

Porque el navegador lee el HTML de arriba hacia abajo. Si los scripts estuvieran en el <head>, el JavaScript intentaría manipular elementos del HTML que todavía no han sido leídos (porque están más abajo), y daría errores. Colocando los scripts al final del <body>, garantizamos que todo el HTML ya fue procesado cuando JavaScript empieza a ejecutarse.

¿Para qué sirve id="appContent"?

<main class="main-content" id="appContent">

El id="appContent" es el "punto de control" de JavaScript para toda la aplicación. El archivo js/app.js busca este elemento y lo usa como la raíz del sistema de pestañas. Cuando JavaScript necesita mostrar u ocultar secciones, sabe que todas están dentro de appContent.

¿Por qué hay tantos archivos JavaScript?

Cada archivo JS tiene una responsabilidad específica:

Archivo Responsabilidad
config.js Variables globales: URL del API, claves de Supabase. Va primero porque los demás lo necesitan.
auth.js Login, logout, verificar si el usuario está autenticado, obtener su rol.
api.js Funciones para comunicarse con el servidor: getProducts(), createOrder(), etc.
cart.js Lógica del carrito: agregar, quitar, calcular totales, guardar en localStorage.
pages/menu.js Todo lo relacionado con la pantalla del menú: renderizar productos, filtros por categoría.
pages/cashier.js El panel del cajero: ver pedidos en tiempo real, cambiar estados.
app.js El coordinador principal. Inicializa todo y gestiona el sistema de pestañas. Va al final porque necesita todos los demás.
Principio de responsabilidad única: Cada archivo JS hace una sola cosa y la hace bien. Esto facilita encontrar bugs, hacer cambios sin romper otras partes, y entender el código. Es una práctica profesional fundamental.

8. El concepto de SPA (Single Page Application)

Cafetería Aroma es una SPA (Single Page Application = Aplicación de Una Sola Página). Entender este concepto es fundamental para comprender cómo funciona todo el sistema.

Web tradicional (Multi-Page Application)

Analogía del libro de papel:
En una web tradicional, cada sección es una página separada. Cuando haces clic en "Ver Menú", el navegador descarga completamente una nueva página (menu.html) desde el servidor. Es como cambiar de hoja en un libro físico: tienes que pasar la página completa.

  • El usuario hace clic en "Menú" → El navegador pide menu.html al servidor
  • El servidor envía menu.html completo
  • El navegador descarga y dibuja toda la página desde cero
  • Hay un momento de "pantalla en blanco" o carga visible

SPA (Single Page Application)

Analogía del control remoto:
En una SPA, toda la aplicación es UNA sola página HTML. No hay recarga. JavaScript muestra y oculta secciones según lo que el usuario quiere ver, como cambiar de canal en la TV: no cambias el televisor, solo cambia el contenido en la misma pantalla.

  • El usuario hace clic en "Menú" → JavaScript oculta el inicio y muestra el menú
  • No hay petición al servidor (o solo una petición de datos, no de HTML)
  • El cambio es instantáneo, sin parpadeos
  • La URL puede cambiar (usando History API) para que el botón "atrás" funcione

¿Qué pasa técnicamente en Aroma?

/* CSS que hace el sistema de tabs funcionar */

/* Por defecto, todas las secciones están ocultas */
.tab-content {
    display: none;   /* ← OCULTO */
}

/* Solo la sección con clase "active" se muestra */
.tab-content.active {
    display: block;  /* ← VISIBLE */
}
/* JavaScript que cambia qué sección está "active" */

function showTab(tabName) {
    // 1. Ocultar TODAS las secciones
    document.querySelectorAll('.tab-content').forEach(tab => {
        tab.classList.remove('active');
    });

    // 2. Mostrar SOLO la sección pedida
    document.getElementById('tab-' + tabName).classList.add('active');

    // 3. Actualizar qué link de la nav está resaltado
    document.querySelectorAll('.nav-link').forEach(link => {
        link.classList.remove('active');
    });
    document.querySelector(`[data-tab="${tabName}"]`).classList.add('active');
}

Ventajas y desventajas de SPA

Multi-Page (web tradicional) SPA (como Aroma)
Velocidad de navegación Más lenta (recarga completa) Muy rápida (solo cambia el contenido)
Experiencia de usuario Parpadeos al cambiar de página Fluida, como una app móvil
Complejidad del código Más simple al principio Más complejo (requiere JavaScript)
SEO (Google) Excelente (Google indexa fácil) Requiere configuración extra
Ejemplo de uso Blogs, sitios de noticias, e-commerce Apps web: Gmail, Google Maps, Trello

¿Por qué Aroma usa SPA?

Cafetería Aroma es una aplicación de gestión (pedidos, panel del cajero, panel del admin), no un blog o tienda para indexar en Google. Lo que necesitamos es velocidad y buena experiencia de usuario: que el cajero pueda cambiar el estado de un pedido sin recargar la página, que el menú filtre productos instantáneamente, que el carrito se actualice en tiempo real. Para eso, una SPA es la elección perfecta.

Resumen de la Guía 01

En esta guía aprendiste:

  • Qué es HTML: Un lenguaje de marcado (no de programación) que define la estructura de una página web. Es el "esqueleto" del sitio.
  • Estructura base: Todo HTML válido comienza con DOCTYPE, tiene head y body. Cada línea tiene un propósito específico.
  • Etiquetas esenciales: Encabezados (h1-h6), párrafos (p), texto (strong, em, span), listas (ul, ol, li), imágenes (img), enlaces (a), contenedores (div, span) y formularios (form, input, button).
  • Atributos HTML: id (único), class (grupo), href, src, alt, data-*, required, placeholder. Son "adjetivos" que modifican el comportamiento y significado de las etiquetas.
  • HTML Semántico: Usar header, nav, main, section, article, aside, footer en vez de div para todo. Mejora accesibilidad, SEO y legibilidad del código.
  • El HTML de Cafetería Aroma: Un sistema de tabs donde todas las secciones viven en el mismo archivo HTML. Los productos no están en el HTML, sino que JavaScript los carga dinámicamente desde la base de datos.
  • SPA (Single Page Application): Toda la app en un solo archivo HTML. JavaScript muestra/oculta secciones según la acción del usuario, sin recargar la página. Más rápido y fluido que las webs tradicionales.
