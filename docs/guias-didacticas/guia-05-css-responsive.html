1. ¿Que es el diseno responsive?

Vivimos en un mundo multi-pantalla. La gente accede a sitios web desde dispositivos con tamanios de pantalla radicalmente diferentes. Un sitio que solo funciona bien en una pantalla de escritorio esta perdiendo la mayoria de sus visitantes.

Los dispositivos mas comunes y sus anchos

/* Anchos tipicos de pantalla en 2025 */

Movil pequeno (iPhone SE):     375px de ancho
Movil mediano (iPhone 14):     390px de ancho
Movil grande (iPhone Plus):    430px de ancho
Tablet (iPad mini):            768px de ancho
Tablet grande (iPad Air):      820px de ancho
Laptop pequena:               1024px de ancho
Laptop estandar:              1280px de ancho
Monitor Full HD:              1920px de ancho
Monitor 4K:                   3840px de ancho

/* La diferencia entre el movil mas chico y el monitor mas grande
   es de 10 veces. Tu diseno debe funcionar en TODO ese rango. */
Dato clave: En 2025, mas del 60% del trafico web mundial proviene de dispositivos moviles. Si tu sitio web no funciona correctamente en un telefono, estas perdiendo a la mayoria de tus usuarios potenciales. Para un negocio como Cafeteria Aroma, donde los clientes podrian ver el menu desde su telefono mientras estan en la cafeteria, esto es critico.

¿Que hace un diseno responsive?

Un diseno responsive se adapta automaticamente al tamanio del dispositivo que lo muestra. No es una version separada del sitio para moviles, es el mismo sitio que cambia su presentacion segun el espacio disponible.

2. El viewport meta tag: la base de todo

Antes de escribir cualquier CSS responsive, necesitas entender el viewport meta tag. Sin el, todo lo demas falla.

¿Que es el viewport?

El "viewport" es el area visible de la pagina web en tu dispositivo. En un movil moderno, los navegadores antiguos simulaban tener 980px de ancho para que los sitios no responsive se vieran completos (muy pequenos, pero completos). Esto es un problema para los sitios responsivos.

<!-- SIEMPRE incluye esto en el <head> de tu HTML -->
<meta name="viewport" content="width=device-width, initial-scale=1.0">
/* Lo que hace cada parte: */

width=device-width
└── El ancho del viewport = ancho fisico del dispositivo
    SIN esto: viewport de 980px (el sitio se ve muy pequeno)
    CON esto: viewport de 375px en un iPhone SE (correcto)

initial-scale=1.0
└── El zoom inicial es 1 (sin zoom)
    SIN esto: el navegador puede hacer zoom out automaticamente
    CON esto: la pagina se muestra al 100% de zoom

/* Comparacion visual: */

SIN meta viewport (incorrecto):          CON meta viewport (correcto):
┌───────────────────────────┐            ┌──────────────────┐
│ [Sitio web               ]│            │ [Sitio web]      │
│ [completo pero muy        ]│            │ [adaptado al     │
│ [chico y dificil de leer] ]│            │ [movil, legible] │
└───────────────────────────┘            └──────────────────┘
   (simula 980px)                           (usa 375px reales)
Regla de oro: Nunca crees un archivo HTML sin el viewport meta tag. Si lo olvidas, tus media queries funcionaran incorrectamente en moviles y todo el trabajo responsive que hagas sera inutil.

3. Mobile-First vs Desktop-First

Hay dos formas de enfocar el diseno responsive. La eleccion afecta como escribes tu CSS.

Desktop-First (disenar para escritorio primero)

/* DESKTOP-FIRST: comienzas con el diseno de escritorio */
.product-grid {
    display: grid;
    grid-template-columns: repeat(4, 1fr); /* 4 columnas en desktop */
    gap: 24px;
}

/* Luego "empequeneces" para pantallas mas chicas con max-width */
@media (max-width: 1024px) {
    .product-grid {
        grid-template-columns: repeat(3, 1fr); /* 3 columnas en tablet */
    }
}

@media (max-width: 768px) {
    .product-grid {
        grid-template-columns: repeat(2, 1fr); /* 2 columnas en tablet chica */
    }
}

@media (max-width: 480px) {
    .product-grid {
        grid-template-columns: 1fr; /* 1 columna en movil */
        gap: 16px;
    }
}

Mobile-First (disenar para movil primero) - Recomendado

/* MOBILE-FIRST: comienzas con el diseno movil */
.product-grid {
    display: grid;
    grid-template-columns: 1fr; /* 1 columna en movil (base) */
    gap: 16px;
}

/* Luego "agrandas" para pantallas mas grandes con min-width */
@media (min-width: 768px) {
    .product-grid {
        grid-template-columns: repeat(2, 1fr); /* 2 columnas en tablet */
        gap: 20px;
    }
}

@media (min-width: 1024px) {
    .product-grid {
        grid-template-columns: repeat(3, 1fr); /* 3 columnas en desktop */
        gap: 24px;
    }
}

@media (min-width: 1280px) {
    .product-grid {
        grid-template-columns: repeat(4, 1fr); /* 4 columnas en pantallas grandes */
    }
}

¿Por que Mobile-First es mejor?

Razon Explicacion
Mayoria de usuarios Mas del 60% del trafico es movil. Disenar para la mayoria primero tiene sentido.
Fuerza a priorizar Con poco espacio en movil, debes elegir que es realmente importante mostrar.
Mejor rendimiento Los moviles descargan el CSS base (mas liviano) y solo los breakpoints necesarios.
Mas facil agregar que quitar Agregar complejidad en pantallas grandes es mas natural que simplificar en pequenas.
Estandar de la industria Frameworks como Bootstrap y Tailwind usan Mobile-First. Es el enfoque moderno.
Cafeteria Aroma usa Mobile-First en todos sus estilos. Cuando revises el archivo styles.css del proyecto, veras que los estilos base son para movil y las media queries usan min-width para ir agregando complejidad.

4. Media Queries: el corazon del responsive

Las media queries permiten aplicar CSS condicional segun las caracteristicas del dispositivo, principalmente el ancho de la pantalla.

Sintaxis basica

@media (condicion) {
    /* CSS que solo se aplica cuando la condicion es verdadera */
}

/* Ejemplos de condiciones: */
@media (min-width: 768px) { ... }   /* Pantalla de 768px o mas */
@media (max-width: 767px) { ... }   /* Pantalla de 767px o menos */
@media (min-width: 768px) and (max-width: 1024px) { ... }  /* Entre 768 y 1024 */
@media (orientation: landscape) { ... }  /* En modo horizontal */
@media (prefers-color-scheme: dark) { ... }  /* Si el SO usa modo oscuro */

Los breakpoints de Cafeteria Aroma

Los "breakpoints" son los puntos de quiebre donde el diseno cambia. Aroma usa estos valores:

/* ===== BREAKPOINTS DE CAFETERIA AROMA ===== */

/* MOVIL: 0px - 767px
   Estilos base (sin media query)
   1 columna, menu hamburguesa, tipografia compacta */

/* TABLET: 768px en adelante */
@media (min-width: 768px) {
    /* 2 columnas de productos, navbar horizontal */
}

/* DESKTOP: 1024px en adelante */
@media (min-width: 1024px) {
    /* 3 columnas, layouts de dos paneles (cajero, admin) */
}

/* WIDE: 1280px en adelante */
@media (min-width: 1280px) {
    /* 4 columnas de productos, contenedor mas ancho */
}

5. Unidades CSS responsive

Ademas de las media queries, las unidades que usas en tu CSS afectan enormemente que tan bien se adapta tu diseno.

Unidades fijas vs relativas

Unidad Tipo Relativa a... Uso recomendado
px Fija Pixeles fisicos Bordes, sombras, espaciados pequenos fijos
% Relativa El elemento padre Anchos fluidos, padding proporcional
rem Relativa Font-size del elemento raiz (<html>) Tipografia escalable, espaciados
em Relativa Font-size del elemento padre Padding/margin en componentes
vw Relativa 1% del ancho del viewport Elementos que ocupan toda la pantalla
vh Relativa 1% del alto del viewport Secciones hero de pantalla completa
vmin Relativa El menor entre vw y vh Elementos cuadrados responsivos

rem vs px: el detalle mas importante

/* Por defecto, el <html> tiene font-size: 16px */
/* 1rem = 16px, 2rem = 32px, 0.875rem = 14px */

/* El problema con px para tipografia: */
h1 { font-size: 32px; } /* Si el usuario aumenta el zoom del navegador, esto NO escala */

/* La solucion con rem: */
h1 { font-size: 2rem; } /* 2 * 16px = 32px, pero SI escala con el zoom del usuario */

/* Esto importa para accesibilidad: algunos usuarios necesitan
   texto mas grande y lo configuran en el navegador */

/* Truco: usar 62.5% para que 1rem = 10px y sea mas facil calcular */
html {
    font-size: 62.5%; /* 1rem = 10px ahora */
}
body {
    font-size: 1.6rem; /* 16px - restablecer el body */
}
h1 { font-size: 3.2rem; } /* 32px - facil de calcular */
p  { font-size: 1.6rem; } /* 16px */

vw y vh en Aroma

/* La seccion hero de la pagina principal */
.hero {
    min-height: 60vh;       /* Al menos 60% de la altura de la pantalla */
    padding: 4rem 1.5rem;   /* rem para que escale con el zoom */
    display: flex;
    align-items: center;
    justify-content: center;
}

/* El panel de cajero ocupa toda la pantalla menos el navbar */
.cashier-layout {
    min-height: calc(100vh - 64px); /* 100% de pantalla - altura del navbar */
    /* calc() permite combinar unidades: 100vh menos 64px fijos */
}

/* El modal de confirmacion cubre toda la pantalla */
.modal-overlay {
    position: fixed;
    top: 0;
    left: 0;
    width: 100vw;   /* 100% del ancho del viewport */
    height: 100vh;  /* 100% del alto del viewport */
    background: rgba(0, 0, 0, 0.5);
    z-index: 1000;
}

El contenedor fluido de Aroma

/* Patron clasico: contenedor con max-width */
.container {
    width: 100%;          /* Siempre 100% del padre */
    max-width: 1200px;    /* Pero no mas de 1200px */
    margin: 0 auto;       /* Centrado horizontalmente */
    padding: 0 1rem;      /* Padding lateral en movil */
}

/* Visualizacion:

En movil (375px):         En tablet (768px):        En desktop (1440px):
┌─────────────────┐       ┌─────────────────────┐   ┌──────────────────────────────┐
│ [contenido]     │       │   [contenido]        │   │    ┌────────────────────┐    │
│ (375px)         │       │   (768px)            │   │    │    [contenido]     │    │

6. Imagenes responsivas

Las imagenes que no se ajustan al tamanio del contenedor pueden romper completamente el diseno en movil. Aprende a manejarlas correctamente.

El truco basico: nunca sobresalir

/* Regla global: las imagenes nunca salen de su contenedor */
img {
    max-width: 100%;  /* La imagen no puede ser mas ancha que su contenedor */
    height: auto;     /* La altura se ajusta proporcionalmente */
}

/* Por que max-width: 100% y no width: 100%?
   - width: 100%   -> la imagen SIEMPRE ocupa todo el ancho (mala para iconos)
   - max-width: 100% -> la imagen nunca SUPERA el ancho, pero puede ser mas chica */

Mantener la proporcion de las imagenes de producto

/* Truco del padding-top para mantener la relacion de aspecto */
.product-image-container {
    width: 100%;
    padding-top: 60%;       /* 60% del ancho = relacion 5:3 */
    position: relative;
    overflow: hidden;
    background: #f0e6d8;    /* Color mientras carga */
    border-radius: 10px 10px 0 0;
}

.product-image {
    position: absolute;
    top: 0;
    left: 0;
    width: 100%;
    height: 100%;
    object-fit: cover;      /* La imagen llena el contenedor sin distorsion */
    object-position: center; /* Centra la imagen dentro del contenedor */
    transition: transform 0.3s ease;
}

.product-card:hover .product-image {
    transform: scale(1.05); /* Zoom suave al hacer hover */
}

/* ¿Como funciona el truco del padding-top?
   - padding-top: 60% se calcula como el 60% del ANCHO del elemento
   - Si el contenedor tiene 300px de ancho: 300 * 0.6 = 180px de alto
   - Si el contenedor tiene 400px de ancho: 400 * 0.6 = 240px de alto
   - Siempre mantiene la proporcion 5:3, sin importar el tamanio */
object-fit: Es como el background-size pero para etiquetas <img>. Con object-fit: cover la imagen siempre llena completamente su contenedor, recortando los bordes si es necesario. Con object-fit: contain la imagen entera se ve, pero pueden aparecer espacios vacios.

9. Transitions y Animations

Las transiciones y animaciones CSS agregan vida a la interfaz de Aroma. Bien usadas mejoran la experiencia del usuario; mal usadas, la empeoran. La clave es la sutileza.

Transitions: cambios suaves entre estados

Una transicion CSS hace que un cambio de estilo (como el color al hacer hover) ocurra gradualmente en lugar de instantaneamente.

/* Sintaxis de transition */
.btn {
    background: #6F4E37;
    color: white;
    padding: 10px 20px;
    border-radius: 8px;
    border: none;
    cursor: pointer;

    /* transition: propiedad duracion funcion-de-timing retraso */
    transition: background 0.2s ease;

    /* Para multiples propiedades: */
    transition: background 0.2s ease,
                transform  0.2s ease,
                box-shadow 0.2s ease;

    /* Para todas las propiedades (usar con cuidado, puede ser lento): */
    transition: all 0.2s ease;
}

.btn:hover {
    background: #4A3426;       /* <- Esta transicion sera suave (0.2s) */
    transform: translateY(-2px);
    box-shadow: 0 6px 16px rgba(74, 52, 38, 0.3);
}
/* Funciones de timing (como "acelera" y "frena" la animacion) */

ease:        empieza lento, acelera, frena (la mas natural, por defecto)
linear:      velocidad constante (para rotaciones infinitas)
ease-in:     empieza lento, termina rapido
ease-out:    empieza rapido, termina lento (para elementos que aparecen)
ease-in-out: empieza y termina lento (para elementos que van y vienen)
cubic-bezier(x1,y1,x2,y2): personalizada (para animaciones avanzadas)

/* Para la mayoria de casos, usa "ease" o "ease-out" */

Transform: mover, escalar y rotar elementos

.elemento {
    /* Mover */
    transform: translateX(10px);    /* 10px hacia la derecha */
    transform: translateY(-5px);    /* 5px hacia arriba */
    transform: translate(10px, -5px); /* Ambos a la vez */

    /* Escalar */
    transform: scale(1.05);         /* 5% mas grande */
    transform: scale(0.95);         /* 5% mas pequeno */
    transform: scaleX(1.1);         /* Solo horizontal */

    /* Rotar */
    transform: rotate(45deg);       /* 45 grados en sentido horario */
    transform: rotate(-90deg);      /* 90 grados en sentido antihorario */

    /* Combinar transformaciones */
    transform: translateY(-4px) scale(1.02);

    /* IMPORTANTE: cuando combinas, el orden importa.
       transform: translateY(-4px) scale(1.02)
       NO es lo mismo que:
       transform: scale(1.02) translateY(-4px)
    */
}

/* Ejemplo en las tarjetas de producto de Aroma */
.product-card {
    transition: transform 0.2s ease, box-shadow 0.2s ease;
}

.product-card:hover {
    transform: translateY(-4px);
    box-shadow: 0 12px 28px rgba(74, 52, 38, 0.15);
}

Keyframe Animations: animaciones complejas

A diferencia de las transitions (que solo van de A a B), las keyframe animations pueden tener multiples estados intermedios.

/* Paso 1: Definir la animacion con @keyframes */
@keyframes girar {
    from { transform: rotate(0deg); }    /* Estado inicial */
    to   { transform: rotate(360deg); }  /* Estado final */
}

/* Paso 2: Aplicar la animacion a un elemento */
.loading-spinner {
    width: 32px;
    height: 32px;
    border: 3px solid #e8d5c0;      /* Circulo incompleto */
    border-top-color: #6F4E37;      /* El "arco" de color cafe */
    border-radius: 50%;

    /* animation: nombre duracion funcion-timing iteraciones */
    animation: girar 0.8s linear infinite;
}
/* Animacion de aparicion de tarjetas en el menu */
@keyframes aparecer {
    from {
        opacity: 0;
        transform: translateY(20px);
    }
    to {
        opacity: 1;
        transform: translateY(0);
    }
}

.product-card {
    animation: aparecer 0.3s ease-out;
    /* Las tarjetas "suben" y aparecen al cargarse */
}

/* Si quieres que cada tarjeta aparezca con un retraso escalonado */
.product-card:nth-child(1) { animation-delay: 0.0s; }
.product-card:nth-child(2) { animation-delay: 0.05s; }
.product-card:nth-child(3) { animation-delay: 0.1s; }
.product-card:nth-child(4) { animation-delay: 0.15s; }
/* Animacion de pulso para el badge del carrito */
@keyframes pulso {
    0%   { transform: scale(1); }
    50%  { transform: scale(1.3); }
    100% { transform: scale(1); }
}

.cart-badge {
    background: #e91e63;
    color: white;
    border-radius: 50%;
    width: 20px;
    height: 20px;
    font-size: 0.75rem;
    display: flex;
    align-items: center;
    justify-content: center;
}

/* Solo pulsa cuando tiene items */
.cart-badge.tiene-items {
    animation: pulso 0.4s ease-out;
}
Cuidado con las animaciones: Demasiadas animaciones distraen al usuario. Usa transiciones suaves y cortas (0.1s - 0.3s) para interacciones. Reserva las animaciones llamativas para momentos importantes (como confirmar un pedido). Evita animar todo.

10. Chrome DevTools para diseno responsive

Chrome DevTools tiene herramientas excelentes para probar y depurar el diseno responsive sin necesitar un dispositivo fisico.

Como abrir DevTools

Opcion 1: Presiona F12
Opcion 2: Click derecho en la pagina -> "Inspeccionar"
Opcion 3: Ctrl + Shift + I (Windows/Linux) o Cmd + Option + I (Mac)

El modo dispositivo (Device Toolbar)

1
Con DevTools abierto, busca el icono de "Toggle device toolbar" (parece un telefono y una tablet). O presiona Ctrl + Shift + M.
2
La pagina cambia a modo responsive. Arriba veras un selector de dispositivos. Elige entre: iPhone SE (375px), iPhone 14 (390px), iPad (768px), etc.
3
Tambien puedes seleccionar "Responsive" y usar el slider para cambiar el ancho manualmente. Observa como las media queries se activan y desactivan al pasar por los breakpoints.
4
Puedes rotar el dispositivo (horizontal/vertical) con el icono de rotacion para probar el modo landscape.
5
Para simular una conexion lenta (como en movil), usa el selector "Network throttling" y elige "Slow 3G" o "Fast 3G".

Inspeccionando el CSS aplicado

/* En el panel "Styles" de DevTools puedes ver: */

1. Todos los estilos CSS aplicados al elemento seleccionado
2. De que archivo y linea vienen esos estilos
3. Que estilos estan siendo sobreescritos (tachados en gris)
4. Cuales media queries estan activas actualmente

/* Para ver solo los estilos que vienen de media queries: */
1. Abre el panel "Sources" -> "media queries bar"
2. O busca en el panel "Styles" las reglas marcadas con el icono de pantalla */
Flujo de trabajo: Mientras desarrollas, ten DevTools abierto en modo responsive. Haz cambios en el CSS, guarda, y verifica en 3 tamanios: 375px (movil), 768px (tablet) y 1280px (desktop). Si funciona en esos tres, funcionara en casi cualquier dispositivo.

Toma el archivo HTML que creaste en el ejercicio de la Guia 04 (el de la grilla de productos) y hazlo completamente responsive.

Grilla de productos Mobile-First: Empieza con 1 columna y agrega columnas en los breakpoints. 
/* Base (movil): 1 columna */
.productos-grid {
    display: grid;
    grid-template-columns: 1fr;
    gap: 16px;
    padding: 16px;
}

    }
}

Prueba en DevTools: Abre el archivo en Chrome, activa el Device Toolbar, y prueba en 375px, 768px y 1280px. Verifica que en cada tamanio se vea correctamente.
5
Agrega transiciones: Agrega transition: transform 0.2s ease, box-shadow 0.2s ease; a las tarjetas y un efecto hover que las suba ligeramente.

Resumen de la Guia 05

  • El diseno responsive adapta el sitio a cualquier tamanio de pantalla; mas del 60% del trafico es movil
  • El viewport meta tag es obligatorio: <meta name="viewport" content="width=device-width, initial-scale=1.0">
  • Mobile-First (con min-width) es el enfoque recomendado: disenar para movil y agregar complejidad para pantallas grandes
  • Las media queries aplican CSS condicional segun el ancho de la pantalla
  • Aroma usa los breakpoints: 768px (tablet), 1024px (desktop), 1280px (wide)
  • Usa rem para tipografia (escala con el zoom), % para anchos fluidos, vw/vh para elementos de pantalla completa
  • Las imagenes deben tener max-width: 100% y usar object-fit: cover para no distorsionarse
  • La navbar de Aroma usa el patron hamburguesa: links ocultos en movil con display: none, visibles en desktop
  • Las transitions hacen los cambios de estado suaves y naturales
  • Las keyframe animations permiten crear efectos como spinners de carga y aparicion de tarjetas
  • Chrome DevTools (F12) tiene un modo de dispositivo para probar el responsive sin dispositivos fisicos
  • En el proyecto, las media queries van al final del archivo styles.css y siguen el patron Mobile-First
