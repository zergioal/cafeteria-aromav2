1. ¿Qué es el DOM?

Analogía: El HTML que escribes en tu editor es como el plano arquitectónico de la cafetería: texto en papel. El DOM es la cafetería construida en la realidad, con mesas, sillas y mostradores que puedes mover, pintar o eliminar. JavaScript interactúa con la cafetería real (el DOM), no con el plano de papel (el HTML estático).

El DOM (Document Object Model) es la representación en memoria que el navegador crea cuando lee tu archivo HTML. Cada etiqueta HTML se convierte en un nodo de un árbol de objetos que JavaScript puede leer y modificar en tiempo real.

HTML (texto plano) DOM (árbol de objetos en memoria)
<html> → document
  <body> → └── body
    <nav> → └── nav
      <a>Menú</a> → └── a (texto: "Menú")
    <main> → └── main
      <div id="app"> → └── div#app

El objeto raíz es siempre document. Desde él puedes llegar a cualquier elemento de la página.

El DOM es vivo: cuando JavaScript modifica el DOM (por ejemplo, agrega un nuevo elemento), el navegador actualiza la página visualmente en tiempo real, sin recargarla. Así funcionan las Single Page Applications (SPA) como Aroma.

2. Seleccionar Elementos del DOM

Antes de poder modificar cualquier elemento, necesitas obtener una referencia a él. Hay varias formas de hacerlo:

// Por ID (devuelve UN elemento o null si no existe)
const navbar = document.getElementById("navbar");
const titulo = document.getElementById("titulo");

// Por selector CSS (devuelve el PRIMERO que coincide)
const boton = document.querySelector(".btn-primary");
const input = document.querySelector("#loginEmail");
const primeraCard = document.querySelector(".product-card");
const spanPrecio = document.querySelector(".product-card .price");

// Por selector CSS (devuelve TODOS - NodeList)
const todasLasCards = document.querySelectorAll(".product-card");
const todosLosBotones = document.querySelectorAll("button[type='submit']");
const navLinks = document.querySelectorAll(".nav-link");

// Iterar una NodeList (similar a un array)
navLinks.forEach(link => {
    console.log(link.textContent);
});
MétodoDevuelveCuándo usarlo
getElementById("id")Un elementoCuando tienes el ID exacto
querySelector(".clase")El primero que coincideSelectores CSS flexibles
querySelectorAll(".clase")NodeList (todos)Cuando necesitas manipular varios
Verifica que el elemento existe: si getElementById o querySelector no encuentran nada, devuelven null. Si intentas acceder a propiedades de null obtendrás un error. Siempre verifica: if (elemento) { ... }.

3. Leer y Modificar el DOM

Una vez que tienes la referencia a un elemento, puedes leer o cambiar cualquier aspecto de él: su texto, sus atributos HTML, sus estilos visuales, sus clases CSS, e incluso su estructura interna.

// Leer y modificar contenido
elemento.textContent = "Nuevo texto";         // Solo texto, sin etiquetas HTML
elemento.innerHTML = "<strong>Texto</strong>"; // Con HTML (cuidado con XSS)

// Leer y modificar atributos
elemento.getAttribute("data-tab")             // Leer el atributo
elemento.setAttribute("data-tab", "menu")     // Escribir o modificar
elemento.removeAttribute("disabled")          // Eliminar atributo
elemento.dataset.tab                           // Leer data-tab (shorthand)
elemento.dataset.productoId                    // Leer data-producto-id

// Leer y modificar estilos (camelCase en lugar de kebab-case)
elemento.style.color = "#6F4E37";
elemento.style.display = "none";           // Ocultar
elemento.style.display = "block";          // Mostrar
elemento.style.backgroundColor = "red";   // background-color → backgroundColor

// Leer y modificar clases CSS (la forma preferida para estilos)
elemento.classList.add("activo");          // Agregar clase
elemento.classList.remove("activo");       // Quitar clase
elemento.classList.toggle("activo");       // Agrega si no tiene, quita si tiene
elemento.classList.contains("activo");     // true/false - verificar si la tiene

// Crear elementos nuevos desde JavaScript
const div = document.createElement("div");
div.className = "product-card";
div.innerHTML = `<h3>Café Latte</h3>`;

// Insertar en el DOM
contenedor.appendChild(div);             // Al final del contenedor
contenedor.prepend(div);                 // Al inicio del contenedor
contenedor.insertBefore(div, referencia); // Antes de otro elemento
div.remove();                            // Eliminar el elemento del DOM

// Navegar el árbol del DOM
elemento.parentElement              // El elemento padre
elemento.children                   // Colección de hijos directos
elemento.nextElementSibling         // El hermano siguiente
elemento.previousElementSibling     // El hermano anterior
classList vs style: Prefiere siempre modificar clases CSS (classList) en lugar de escribir estilos directamente (element.style). De esta forma mantienes el diseño en el CSS y la lógica en JavaScript, separando responsabilidades. Solo usa element.style para valores calculados dinámicamente (ej: posición X/Y de un elemento arrastrable).

4. Eventos – Respondiendo a las Acciones del Usuario

Los eventos son el corazón de la interactividad. Un evento ocurre cuando el usuario hace algo (click, escribe, mueve el mouse) o cuando el navegador termina de cargar algo. Con addEventListener le dices a JavaScript: "cuando ocurra este evento en este elemento, ejecuta esta función".

// Estructura básica de addEventListener
elemento.addEventListener("nombreDelEvento", funcionHandler);

// Con función inline
boton.addEventListener("click", function(evento) {
    console.log("¡Botón clickeado!");
    console.log(evento.target); // El elemento exacto que fue clickeado
    console.log(evento.type);   // "click"
});

// Con arrow function (lo más común en Aroma)
boton.addEventListener("click", (e) => {
    e.preventDefault();   // Evitar el comportamiento por defecto del navegador
    e.stopPropagation();  // Evitar que el evento "suba" hacia los elementos padre
    // Tu código aquí
});

// Catálogo de eventos más usados en Aroma:
elemento.addEventListener("click", handler);          // Click del mouse
elemento.addEventListener("dblclick", handler);       // Doble click
elemento.addEventListener("mouseover", handler);      // Mouse entra al área
elemento.addEventListener("mouseout", handler);       // Mouse sale del área
elemento.addEventListener("keydown", handler);        // Tecla presionada
elemento.addEventListener("keyup", handler);          // Tecla soltada
input.addEventListener("input", handler);             // Cada vez que escribe
input.addEventListener("change", handler);            // Al perder el foco con cambio
form.addEventListener("submit", handler);             // Envío del formulario
window.addEventListener("load", handler);             // Toda la página cargada
document.addEventListener("DOMContentLoaded", handler); // Solo el DOM listo

El objeto event

Cada handler recibe automáticamente un objeto con información sobre el evento que ocurrió. Lo más útil:

boton.addEventListener("click", (e) => {
    e.target            // El elemento exacto donde se hizo click
    e.currentTarget     // El elemento al que se agregó el listener
    e.preventDefault()  // Cancela la acción predeterminada (ej: envío de form)
    e.key               // En keydown: la tecla presionada ("Enter", "Escape", etc.)
});

input.addEventListener("keydown", (e) => {
    if (e.key === "Enter") {
        buscarProducto();
    }
    if (e.key === "Escape") {
        cerrarModal();
    }
});
¿Cuándo usar e.preventDefault()? Úsalo cuando el elemento tiene un comportamiento nativo que no quieres. Ejemplos: en un <a href="#"> para que no cambie la URL; en un <form> para evitar que recargue la página al enviarlo; en drag-and-drop para permitir soltar elementos.

5. El Sistema de Navegación de Aroma (app.js)

Archivo real del proyecto: frontend/js/app.js

Este módulo implementa el sistema de navegación tipo SPA (Single Page Application): cambia el contenido visible sin recargar la página. Analiza cada parte con cuidado, ya que usa todos los conceptos que acabas de aprender.

// frontend/js/app.js - Sistema de navegación SPA

const App = {
    // La pestaña que se está mostrando actualmente
    currentTab: 'home',

    // Inicializar la aplicación
    init() {
        // Esperar a que el DOM esté completamente listo antes de ejecutar nada
        document.addEventListener('DOMContentLoaded', () => {
            this.setupNavigation();
            this.showTab('home');
        });
    },

    // Configurar los clicks de la barra de navegación
    setupNavigation() {
        // Seleccionar todos los links de navegación
        const navLinks = document.querySelectorAll('.nav-link');

        // Para cada link, agregar un listener de click
        navLinks.forEach(link => {
            link.addEventListener('click', (e) => {
                e.preventDefault(); // Evitar que el navegador cambie la URL

                const tab = link.dataset.tab; // Leer el atributo data-tab="menu"
                this.showTab(tab);
            });
        });
    },

    // Mostrar una pestaña específica y ocultar todas las demás
    showTab(tabName) {
        // Paso 1: Ocultar TODAS las pestañas
        document.querySelectorAll('.tab-content').forEach(tab => {
            tab.classList.remove('active');
        });

        // Paso 2: Quitar la clase "active" de TODOS los links
        document.querySelectorAll('.nav-link').forEach(link => {
            link.classList.remove('active');
        });

        // Paso 3: Mostrar SOLO la pestaña seleccionada
        const tabElement = document.getElementById(`tab-${tabName}`);
        if (tabElement) {
            tabElement.classList.add('active');
        }

        // Paso 4: Marcar el link correspondiente como activo
        const activeLink = document.querySelector(`[data-tab="${tabName}"]`);
        if (activeLink) {
            activeLink.classList.add('active');
        }

        // Paso 5: Guardar la pestaña actual en el estado del objeto
        this.currentTab = tabName;
    }
};

// Iniciar la aplicación
App.init();

Analizando cada decisión del código

¿Por qué usamos un objeto App? (Module Pattern)

En lugar de tener funciones y variables sueltas, las encapsulamos dentro de un objeto. Esto tiene varias ventajas:

  • Evita contaminar el espacio de nombres global (no hay decenas de funciones y variables sueltas).
  • Agrupa el código relacionado en un solo lugar (todo lo de la navegación está en App).
  • Los métodos pueden compartir estado interno usando this.currentTab.
  • Queda claro de dónde viene cada función: App.showTab() es más descriptivo que showTab().

¿Qué hace DOMContentLoaded?

Si JavaScript intenta seleccionar un elemento antes de que el HTML lo haya creado, obtendrá null. El evento DOMContentLoaded se dispara en el momento exacto en que el navegador terminó de parsear el HTML completo y construyó el árbol DOM, pero antes de cargar imágenes o estilos externos. Es el momento seguro para empezar a manipular el DOM.

¿Por qué e.preventDefault() en los links de navegación?

Los links de navegación en Aroma tienen atributos data-tab pero no un href real, o tienen href="#". Sin preventDefault(), al hacer click el navegador intentaría navegar a otra URL (o hacer scroll al inicio de la página con #), recargando la aplicación y perdiendo el estado del carrito y del usuario logueado.

¿Cómo funciona dataset.tab?

<!-- En el HTML -->
<a class="nav-link" data-tab="menu">Menú</a>
<a class="nav-link" data-tab="carrito">Carrito</a>

// En JavaScript:
link.dataset.tab      // "menu" o "carrito"
// dataset convierte automáticamente kebab-case a camelCase:
// data-producto-id → dataset.productoId

El patrón "ocultar todos, mostrar uno"

Este es un patrón fundamental en interfaces SPA. En lugar de rastrear qué estaba visible antes, simplemente:

  1. Quita la clase active de todos los elementos del grupo.
  2. Agrega la clase active solo al elemento deseado.

Es simple, predecible y no deja estados inconsistentes.

6. Event Delegation – Un Solo Listener para Muchos Elementos

Imagina que el menú de Aroma tiene 50 productos. Si agregas un addEventListener a cada botón "Agregar", estás creando 50 funciones en memoria. Además, cuando los productos se cargan dinámicamente (desde la API), los botones no existen todavía cuando el código se ejecuta.

El problema: los listeners solo se pueden agregar a elementos que ya existen en el DOM. Los elementos creados dinámicamente después (con innerHTML) no tienen los listeners.

La solución: Event Delegation. En lugar de escuchar el evento en cada botón hijo, escuchamos en el elemento padre que SÍ existe desde el principio. Cuando el usuario hace click en cualquier lugar del padre, el evento "burbujea" hacia arriba y el padre lo captura. Luego verificamos si el click fue en el elemento específico que nos interesa.

// MAL - 50 listeners, no funciona con elementos dinámicos
document.querySelectorAll('.btn-agregar').forEach(btn => {
    btn.addEventListener('click', agregarAlCarrito); // Se agrega a elementos existentes
});
// Los botones que se creen después con innerHTML NO tendrán este listener

// BIEN - 1 solo listener en el contenedor padre (que siempre existe)
document.getElementById('productGrid').addEventListener('click', (e) => {
    // Verificar si el click fue exactamente en un botón "agregar"
    if (e.target.classList.contains('btn-agregar')) {
        // Leer los datos del producto desde los atributos data-*
        const productoId = e.target.dataset.id;
        const productoNombre = e.target.dataset.nombre;
        const productoPrecio = parseFloat(e.target.dataset.precio);

        agregarAlCarrito(productoId, productoNombre, productoPrecio);
    }

    // También podemos manejar otros clicks en el mismo listener
    if (e.target.classList.contains('btn-detalle')) {
        const productoId = e.target.dataset.id;
        verDetalleProducto(productoId);
    }
});
Event Delegation funciona gracias al "bubbling": cuando haces click en un botón, el evento no solo se dispara en el botón, sino que después "sube" (burbujea) por todos sus ancestros: el div contenedor, el main, el body, hasta llegar al document. El listener en el padre captura el evento aunque el click original fue en el hijo.

7. Generar HTML Dinámico de Forma Segura

Una de las habilidades más importantes en el desarrollo frontend es construir HTML desde datos JavaScript. Así es como Aroma muestra los productos que vienen del servidor.

// Función para crear la tarjeta HTML de un producto
function crearTarjetaProducto(producto) {
    // Usamos template literals para construir el HTML como string
    return `
        <div class="product-card" data-id="${producto.id}">
            <div class="product-image-container">
                <img
                    src="${producto.imagen_url || 'img/placeholder.jpg'}"
                    alt="${escHtml(producto.nombre)}"
                    class="product-image"
                    loading="lazy"
                >
            </div>
            <div class="product-info">
                <h3 class="product-name">${escHtml(producto.nombre)}</h3>
                <p class="product-desc">${escHtml(producto.descripcion || '')}</p>
                <div class="product-footer">
                    <span class="product-price">Bs. ${producto.precio.toFixed(2)}</span>
                    <button
                        class="btn-agregar"
                        data-id="${producto.id}"
                        data-nombre="${escHtml(producto.nombre)}"
                        data-precio="${producto.precio}"
                    >
                        + Agregar
                    </button>
                </div>
            </div>
        </div>
    `;
}

// Función de seguridad: prevenir XSS escapando caracteres HTML peligrosos
function escHtml(str) {
    if (!str) return '';
    return str
        .replace(/&/g, '&amp;')
        .replace(/</g, '&lt;')
        .replace(/>/g, '&gt;')
        .replace(/"/g, '&quot;')
        .replace(/'/g, '&#039;');
}

// Renderizar TODOS los productos en el contenedor
function renderizarProductos(productos) {
    const grid = document.getElementById('productGrid');

    if (productos.length === 0) {
        grid.innerHTML = `
            <div class="empty-state">
                <p>No hay productos disponibles</p>
            </div>
        `;
        return; // Salir de la función aquí
    }

    // map() convierte cada producto en string HTML, join() los une en uno solo
    grid.innerHTML = productos.map(crearTarjetaProducto).join('');
}
XSS (Cross-Site Scripting) – Por qué es crítico usar escHtml(): Si un atacante logra guardar en la base de datos un nombre de producto como <script>robarContrasenas()</script>, y tú lo insertas directamente con innerHTML, ese script malicioso se ejecutará en el navegador de todos los usuarios que vean el menú. La función escHtml() convierte los caracteres <, >, " y & en sus equivalentes seguros (&lt;, etc.), que el navegador mostrará como texto literal, no como código.

8. Toast Notifications – Mensajes Emergentes

Las notificaciones "toast" son pequeños mensajes que aparecen temporalmente (generalmente en una esquina de la pantalla) para confirmar acciones o mostrar errores, sin interrumpir al usuario con un alert() bloqueante.

// Sistema de notificaciones de Aroma
function mostrarToast(mensaje, tipo = 'info', duracion = 3000) {
    // 1. Crear el elemento DOM del toast
    const toast = document.createElement('div');
    toast.className = `toast toast-${tipo}`;
    toast.innerHTML = `
        <span class="toast-icon">${getToastIcon(tipo)}</span>
        <span class="toast-mensaje">${escHtml(mensaje)}</span>
        <button class="toast-close" onclick="this.parentElement.remove()">×</button>
    `;

    // 2. Agregar al contenedor (o crear el contenedor si no existe)
    const contenedor = document.getElementById('toastContainer')
        || crearContenedorToast();
    contenedor.appendChild(toast);

    // 3. Pequeño delay para que el CSS pueda animar la entrada
    setTimeout(() => toast.classList.add('visible'), 10);

    // 4. Auto-eliminar después de la duración especificada
    setTimeout(() => {
        toast.classList.remove('visible');           // Animar salida
        setTimeout(() => toast.remove(), 300);      // Eliminar del DOM tras animación
    }, duracion);
}

function getToastIcon(tipo) {
    const iconos = {
        success: '✅',
        error: '❌',
        warning: '⚠️',
        info: 'ℹ️'
    };
    return iconos[tipo] || iconos.info;
}

// Ejemplos de uso en Aroma:
mostrarToast('Producto agregado al carrito', 'success');
mostrarToast('Debes iniciar sesión para ordenar', 'warning');
mostrarToast('Error al cargar productos', 'error');
mostrarToast('Bienvenido a Cafetería Aroma', 'info');

Esta función es un buen ejemplo de múltiples conceptos combinados:

  • Parámetros por defecto: tipo = 'info' y duracion = 3000 hacen que los 2 últimos parámetros sean opcionales.
  • createElement e innerHTML: para construir la notificación dinámicamente.
  • classList.add/remove: para controlar las animaciones CSS.
  • setTimeout: para ejecutar código después de un delay (en milisegundos).
  • Encadenamiento de setTimeouts: primero se espera la duración del toast, luego se espera la animación de salida (300ms), luego se elimina del DOM.

Resumen de la Guía 07

  • El DOM es la representación en memoria del HTML. JavaScript lo manipula en tiempo real para crear interfaces dinámicas sin recargar la página.
  • Selecciona elementos con getElementById, querySelector (primero que coincide) y querySelectorAll (todos los que coinciden).
  • Modifica clases con classList.add/remove/toggle/contains; es más limpio que escribir estilos directamente.
  • Usa addEventListener para responder a eventos. El objeto event que recibe el handler tiene información sobre lo que ocurrió.
  • e.preventDefault() cancela el comportamiento predeterminado del navegador (necesario en links de navegación SPA y formularios).
  • DOMContentLoaded es el evento que indica que el DOM está listo para ser manipulado de forma segura.
  • El Module Pattern (const App = { métodos... }) organiza el código evitando variables globales sueltas.
  • dataset.nombreAtributo permite leer atributos data-* del HTML de forma limpia.
  • Event Delegation: un solo listener en el padre maneja eventos de todos sus hijos, incluso los creados dinámicamente.
  • Usa siempre escHtml() al insertar datos del usuario o del servidor con innerHTML para prevenir ataques XSS.
  • Las notificaciones toast combinan createElement, classList y setTimeout para mostrar mensajes animados sin bloquear al usuario.
